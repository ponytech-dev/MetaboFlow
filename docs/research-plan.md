## 1. 科研愿景

### 1.1 学术定位：不仅是产品，更是科研平台

MetaboFlow 的学术定位是**代谢组学方法学研究的基础设施平台**，而非单纯的软件工具。这一定位建立在以下核心观察上：

代谢组学领域当前面临深刻的**方法学危机**：同一数据集经四个主流引擎（XCMS、MZmine、MS-DIAL、Compound Discoverer）处理后，特征共识率仅 8%（Analytica Chimica Acta 2024）；244 项临床研究的荟萃分析显示，85% 的"显著差异代谢物"实为统计噪声；代谢物注释率长期停留在 10% 以下。这些问题不是工程问题，而是方法学问题，需要算法层面的根本性突破。

MetaboFlow 的双重身份：
- **产品侧**：引擎聚合平台，让非生信用户完成全流程分析
- **科研侧**：方法学创新的验证平台，系统性积累多引擎基准数据，推动算法创新发表

两者协同而非竞争：产品积累的大规模多引擎运行数据，是方法学研究最宝贵的素材；方法学论文发表，为产品建立科学可信度和社区影响力。

```
产品积累数据
    ↓
多引擎基准测试 → 发现方法学 Gap → 算法创新研究
    ↓                                    ↓
```

### 2.0 原创杀手级算法（S 级）

S 级方向的特征：**从代谢组学自身的物理/化学/生物学第一性原理出发**，而非从其他领域的方法出发寻找应用场景。这些算法不存在"为什么不直接用现有方法"的反驳空间，因为它们解决的是代谢组学特有的、现有方法论框架根本无法触及的问题。

---

#### 方向 S1：生成式离子反卷积（Generative Ion Deconvolution, GID）

**解决什么问题**

当前所有代谢组学引擎（XCMS、MZmine、MS-DIAL、asari）都遵循同一个范式：**自底向上的特征检测**——先在原始数据中找到所有峰（特征），再尝试将这些特征归属到分子。这个范式有一个根本性缺陷：**特征膨胀**。

一个分子进入质谱仪后，不是产生一个特征，而是通过离子化物理过程产生 5-10 个特征：
- 单电荷加合物：[M+H]⁺, [M+Na]⁺, [M+K]⁺, [M+NH₄]⁺
- 多电荷：[M+2H]²⁺
- 源内碎裂（ISF）：本应在 MS2 中产生的碎片离子出现在 MS1 中
- 同位素峰：¹³C, ³⁴S 等
- 二聚体：[2M+H]⁺, [2M+Na]⁺

结果：100 个真实分子在一个典型 LC-MS 实验中产生 500-1000 个"特征"，其中大量是**同一分子的不同离子形式**。现有引擎无法区分"这 7 个特征其实是同一个分子"的情况，用户看到的特征矩阵里 80% 都是重复信息。这直接导致：
- 统计分析中的多重检验惩罚被人为放大（10000 个特征 vs 1000 个真实分子）
- 通路富集分析中同一分子被计数多次，扭曲结果
- "85% 被判为噪声"的结论部分来自特征膨胀

**现有方法为什么不够**

现有工具（如 CAMERA、MS-FLO、CliqueMS）尝试通过**规则匹配**（已知加合物质量差 + 共洗脱检查）来分组特征。但这些方法是**后处理式的打补丁**——先让特征膨胀发生，再尝试事后清理。问题：
- 规则不完整：ISF 碎片无规律可循，无法用质量差规则捕获
- 错误级联：先检测到 7 个峰，每个都有定量误差，然后尝试把它们合并，误差叠加
- 无法处理重叠：如果两个分子的某个加合物 m/z 相近，规则匹配无法区分

**GID 的核心思路：倒过来**

GID 不是"检测特征→归属到分子"，而是直接建模**分子→离子群**的生成过程：

```
传统范式（Bottom-Up）：
  原始数据 → 峰检测 → 特征列表 → 加合物归属（打补丁）→ 分子列表

GID 范式（Top-Down）：
  假设存在分子 M → 预测 M 应该产生哪些离子 → 在原始数据中寻找这些离子的证据 → 评分
```

**具体技术路线**

1. **离子化物理模型**：给定一个分子（SMILES/分子式），预测它在 ESI 源中可能产生的所有离子种类及其相对强度
   - 加合物概率模型：基于 HMDB/MoNA 大规模统计，不同结构类别（脂质/氨基酸/有机酸等）的加合物分布不同
   - ISF 概率模型：基于分子内弱键（酯键、酰胺键）预测可能的源内碎裂
   - 同位素模式：精确化学式 → 精确同位素分布（已有成熟算法 IsoSpec）

2. **贝叶斯反向推理**：将问题形式化为概率推理
   - 观测：原始 LC-MS 数据（m/z × RT × intensity 三维张量）
   - 隐变量：存在哪些分子，每个分子的浓度是多少
   - 似然函数：P(观测数据 | 分子列表, 浓度) = Π P(每个分子生成的离子群 | 分子, 浓度)
   - 推理：变分推断（Variational Inference）或 MCMC 采样

3. **候选分子来源**：
   - 已知代谢物数据库（HMDB 220K 条目，KEGG，MetaCyc）
   - 候选分子式枚举（Seven Golden Rules + RDBE 约束）
   - DreaMS 嵌入空间中的近邻搜索

4. **迭代求解**：
   - 初始化：用传统峰检测（centWave）提供初始候选
   - E 步：给定当前分子列表，计算每个数据点归属于哪个分子的哪个离子
   - M 步：更新分子列表和浓度（添加/删除分子，调整浓度）
   - 收敛判据：残差数据中无显著峰（所有可解释的信号已被归属）

**创新点（为什么这是原创的）**

- **范式创新**：从"检测→归属"翻转为"假设→验证"，是代谢组学数据处理 20 年来首次范式转变
- **原生于代谢组学**：离子化物理模型是质谱特有的，不是从其他领域移植的方法
- **解决根本问题**：特征膨胀不是被后处理清理，而是从一开始就不产生

**技术可行性评分**：7/10
- 挑战：候选分子搜索空间大，计算成本高（需要高效剪枝策略）
- 有利：核心算法组件（变分推断、IsoSpec、加合物规则）均已成熟
- 降低风险的策略：先在**已知混合标准品**数据上验证（分子列表已知，只需推断浓度），再推广到未知样品

**创新潜力评分**：10/10
- 代谢组学领域没有任何工具采用生成模型范式
- 最接近的工作是 mzMine 4 的 IIMN（Ion Identity Molecular Networking），但 IIMN 仍是后处理式的规则匹配，不是概率生成模型

**问题清晰度评分**：10/10
- 特征膨胀是每个代谢组学用户都直接面对的问题
- 可量化评估：用已知标准品数据集，比较 GID 输出的分子列表 vs 传统流程 + CAMERA 的去冗余结果

**发表潜力**
- 目标期刊：**Nature Methods**（范式创新级别的工作）
- 核心卖点：首个将代谢组学数据处理从"特征检测"范式转向"分子检测"范式的方法
- 风险：实现难度高，但即使部分成功（如仅在标准品混合物上有效），仍然可以在 **Analytical Chemistry** 发表概念验证

**预计投入**：6-8 个月（含算法开发 + 标准品验证 + 撰写）

**前置依赖**：MetaboFlow 峰检测引擎架构（提供初始化）+ HMDB 分子数据库集成

#### 方向 S2：共检测约束注释（Co-detection Constrained Annotation, COCA）

**解决什么问题**

代谢物注释（"这个特征是什么分子"）是代谢组学最核心也最薄弱的环节。当前注释体系依赖两类独立证据：
1. **谱图匹配**：将未知谱图与数据库谱图比对（matchms, GNPS）
2. **反应传播**：通过代谢网络拓扑从已知注释推断相邻代谢物（FBMN, MetDNA）

问题：这两类证据都有严重局限：
- 谱图匹配覆盖率低：MoNA/GNPS 仅覆盖 ~5% 的已知代谢物
- 反应传播误差累积：每传播一步，错误率指数增长（MetDNA 论文自身数据显示 3 步传播后准确率降至 <50%）

**COCA 的核心洞察：第三类独立证据**

代谢组学有一个被忽视的信息来源：**跨样本共检测模式**。

如果 MetaboFlow 积累了大量的多引擎运行结果（这正是平台产品的核心资产），那么对每个特征，我们知道它"在哪些样本中被检测到"。这个共检测模式蕴含着强大的约束信息：

- **生化约束**：同一代谢通路上的代谢物倾向于共检测（甘油三酯水解通路上的 TG/DG/MG 同时出现或同时消失）
- **组织约束**：肝脏特异性代谢物（胆汁酸）倾向于在肝脏样本中共检测，不出现在脑组织中
- **疾病约束**：糖尿病特征代谢物（葡萄糖/HbA1c 相关代谢物）在患者样本中共检测

这个信息是谱图匹配和反应传播都无法提供的**第三维独立证据**。

**具体技术路线**

1. **共检测矩阵构建**：
   - 从 MetaboFlow 平台历史数据中提取"特征 × 样本"的检测矩阵（binary: detected/not detected）
   - 计算所有特征对的共检测系数（Jaccard index 或 phi correlation）
   - 构建共检测网络：节点=特征，边=显著共检测关系

2. **代谢网络先验**：
   - 从 KEGG/Reactome 获取代谢反应网络
   - 预测：如果特征 A 被注释为代谢物 X，而 X 在代谢网络中与 Y 相邻，那么与 A 共检测最强的特征中应该有被注释为 Y 的
   - 这构成一个**一致性约束**：注释结果必须与共检测模式在代谢网络拓扑上一致

3. **概率图模型整合**：
   - 变量：每个特征的注释（从候选列表中选择一个分子身份）
   - 因子 1：谱图匹配得分（传统证据）
   - 因子 2：反应传播得分（传统证据）
   - 因子 3（新）：共检测一致性得分——如果特征 A 注释为 X，特征 B 注释为 Y，且 A-B 共检测强度高，那么 X-Y 在代谢网络中的距离应该近
   - 使用信念传播（Belief Propagation）或变分推断求解全局最优注释

4. **自举验证**：
   - 从高置信度注释出发（Level 1 标准品匹配），检查它们的共检测模式是否符合代谢网络预期
   - 如果不符合 → 质疑注释质量
   - 如果符合 → 用这些锚点传播约束到低置信度特征

**创新点（为什么这是原创的）**

- **数据平台原生算法**：只有聚合了大量样本数据的平台才能计算有意义的共检测模式。MetaboFlow 的平台架构是 COCA 的天然基础设施
- **第三类独立证据**：现有注释体系从未系统性利用跨样本共检测信息
- **全局一致性优化**：传统方法逐特征独立注释，COCA 将所有特征的注释作为一个联合优化问题

**现实验证**

这个思路不是纯理论猜想，有间接但强力的支持证据：
- GNPS 的分子网络本质上就是在利用"相似特征倾向于是相关分子"这个信息，但它只用了谱图相似性（MS2），没有用共检测模式（MS1 层面跨样本的出现/缺失）
- 蛋白质组学的 MaxLFQ 算法已经证明"跨样本的共检测一致性"可以大幅提升定量准确度
- 单细胞组学的 dropout imputation 方法也依赖共检测模式推断缺失值

**技术可行性评分**：8/10
- 有利：概率图模型和信念传播有成熟实现（pgmpy, libDAI）
- 有利：共检测矩阵从现有数据直接计算，无需新数据采集
- 挑战：需要足够多的样本数据才能计算稳健的共检测统计量（但 MetaboFlow 作为平台恰好能积累这些数据）

**创新潜力评分**：10/10
- 文献空白：没有任何代谢组学注释工具利用跨样本共检测模式
- 独特优势：只有多引擎聚合平台才有数据基础实现此算法，形成竞争壁垒

**问题清晰度评分**：9/10
- 注释率低（<10%）是公认的核心痛点
- 评估方法明确：已知注释的数据集上，COCA 注释率和准确率 vs matchms + MetDNA baseline

**发表潜力**
- 目标期刊：**Nature Methods**
- 核心卖点：首次系统性利用跨样本共检测模式作为代谢物注释的独立证据源
- 与 MetaboFlow 平台的协同：论文的核心数据直接来自平台用户积累的分析结果

**预计投入**：5-7 个月（含数据积累 + 算法开发 + 验证 + 撰写）

**前置依赖**：MetaboFlow 平台有足够的历史分析数据（≥500 个样本的共检测矩阵），注释引擎集成完成

#### 方向 A：DreaMS 式自监督谱图预训练 + RAG 注释

**解决什么问题**

代谢物注释的核心瓶颈是**数据库覆盖率**：MoNA/GNPS 只有 ~60 万条 MS2 谱图，覆盖不到 5% 的已知代谢物。对于数据库中没有的代谢物，谱图匹配直接失效。

DreaMS（Nature Biotechnology 2025）用自监督学习在 ~5 亿条谱图上预训练了谱图 Transformer，学到了"化学空间的通用语言"——但 DreaMS 本身不做注释。MetaBench（arXiv 2025）发现直接用 LLM 做注释时 grounding 准确率仅 0.87%，因为 LLM 不知道具体的化学事实。

**为什么 RAG 是正确的解法**

RAG 精确解决了 DreaMS 和 LLM 的互补性问题：
- DreaMS 有强大的谱图表征能力，但不知道具体化学知识
- LLM 有推理能力，但缺乏专业 grounding
- RAG = DreaMS 表征 + 知识库检索 + LLM 推理，三者组合恰好覆盖单一方法的盲区

**具体技术路线**

1. **谱图嵌入层**：基于 DreaMS 开源模型微调，输入 MS2 谱图 → 输出 512 维嵌入向量
2. **知识库构建**：
   - 向量化 HMDB 220K 条目的谱图+元数据（分子式、分类、生物来源、已知代谢通路）
   - 向量化 MassBank/MoNA 所有已注释谱图
   - 向量化代谢通路文本描述（KEGG Pathway 文本 + reaction descriptions）
   - 全部存入 Qdrant（MetaboFlow 已有基础设施）
3. **推理链**：
   ```
   输入谱图 → DreaMS 嵌入 → Qdrant Top-K 检索 →
   组装 prompt（谱图特征 + K 个候选 + 化学约束）→
   LLM 推理排序 → 输出：候选列表 + 置信度 + 推理路径
   ```
4. **关键创新**：推理路径可追溯（用户可以看到"为什么选这个候选"），解决了现有 ML 注释方法的黑盒问题

**与现有方法的真实对比优势**

| 能力 | matchms (谱图匹配) | MetDNA (反应传播) | SIRIUS (分子式预测) | DreaMS+RAG |
|------|-------------------|------------------|-------------------|-----------|
| 数据库外的分子 | ✗ | 有限（1-2 步） | 分子式级别 | ✓（嵌入空间邻近搜索） |
| 推理可解释性 | 相似度分数 | 通路路径 | 候选排序分 | **完整推理链** |
| 上下文感知 | ✗ | ✗ | ✗ | ✓（通路/组织类型/疾病背景） |

#### 方向 B：TDA/持久同调用于 2D 峰检测

**解决什么问题**

所有现有峰检测算法（centWave、ADAP、MassCube）的核心逻辑都是**阈值切割**：在 EIC 上设定信噪比阈值，高于阈值的区域视为峰。这有两个根本缺陷：
1. **阈值是全局的**：信噪比高的区域和信噪比低的区域用同一个阈值，要么漏掉低丰度真实峰，要么引入高噪声区域的假阳性
2. **无法处理重叠峰**：两个部分重叠的色谱峰被检测为一个，centWave 的高斯拟合假设在这里失效

**为什么 TDA 是正确的解法**

持续同调（Persistent Homology）的核心思想：**不需要设定阈值**。它从最高强度开始逐渐降低阈值，追踪"拓扑连通分量"的出生和死亡：
- **真实峰**：在高阈值时出生，在低阈值时仍然存活 = 高持续性
- **噪声**：在某个阈值时短暂出现又消失 = 低持续性
- 持续性条码（barcode）天然区分信号和噪声，不需要人为设定阈值

2024 年 Analytica Chimica Acta 已在 GC-IMS 2D 数据上验证了此方法的可行性，但 **LC-MS 的 2D 峰检测（RT × m/z）完全空白**。

**具体技术路线**

1. 将 LC-MS 的 m/z × RT × intensity 三维数据视为二维地形图（intensity 为高度）
2. 使用 Vietoris-Rips 复合体构建持续同调，追踪 H₀（连通分量=峰）和 H₁（环=重叠峰之间的鞍点）
3. 利用持续性条码的 birth-death 信息：
   - birth-death 距离大 → 真实峰（高持续性）
   - birth-death 距离小 → 噪声
   - 两个相邻连通分量的鞍点 → 重叠峰的分界线
4. 与 centWave/ADAP/MassCube 在标准品数据集上系统性对比

**真实优势**
- 免参数：不需要调 SNR 阈值、峰宽范围等参数（代谢组学的"参数地狱"痛点）
- 重叠峰分离：拓扑方法天然处理连通分量的合并/分裂
- 数学严格性：持续同调的稳定性定理保证结果对小扰动鲁棒

**技术可行性评分**：8/10（`giotto-tda` 库成熟，GC-IMS 先例提供方法论参考）
**创新潜力评分**：8/10（LC-MS 应用是空白）
**问题清晰度评分**：9/10（峰检测是每个用户必经步骤，参数调优是第一大痛点）

**发表潜力**：**Nature Methods**（若性能显著超越 centWave）或 **Analytical Chemistry**

**预计投入**：4-5 个月

**前置依赖**：峰检测引擎架构完成（Phase 1），有标准品测试数据集

#### 方向 C：遥感端元提取 → MSI 分子解卷积

**解决什么问题**

质谱成像（MSI）中每个像素是多种代谢物的空间混合信号。当前 MSI 分析方法（SCiLS、CARDINAL）主要使用聚类（k-means、NMF）将像素分组为"组织区域"，但这告诉你的是"哪些像素相似"，而不是"每个像素里有哪些分子，各占多少"。

**为什么遥感解混不是表面类比**

遥感高光谱图像和 MSI 数据在数学上是**精确同构**的：

| 概念 | 高光谱遥感 | 质谱成像 |
|------|----------|---------|
| 像素 | 地面 30m×30m 区域 | 组织 20μm×20μm 区域 |
| 光谱维度 | 200-400 个波长通道 | 1000-50000 个 m/z 通道 |
| 端元 | 纯物质反射光谱（水/植被/土壤） | 纯代谢物质谱（ATP/乳酸/胆固醇） |
| 混合模型 | 线性混合假设 | 线性混合假设 |
| 目标 | 端元提取 + 丰度估计 | 代谢物鉴定 + 浓度空间分布 |

遥感领域有 20 年的算法积累（VCA、SUNSAL、深度学习 autoencoder unmixing），这些算法可以**几乎不修改**地直接用于 MSI 数据，因为底层数学模型完全一致。

这不是"灵感来自遥感的方法"，而是"在精确同构的数据上应用已验证的算法"。

**具体技术路线**

1. 将 imzML 格式 MSI 数据转为"像素×m/z"矩阵（与高光谱"像素×波长"矩阵同构）
2. 端元提取：VCA（Vertex Component Analysis）找到纯像素 → 提取端元谱图
3. 丰度估计：SUNSAL（Sparse Unmixing by variable Splitting and Augmented Lagrangian）估计每个像素中各端元的丰度
4. 端元注释：将提取的端元谱图与 MassBank/HMDB 进行匹配 → 得到"纯代谢物"身份
5. 输出：每种代谢物的空间分布热图（不是"相似像素的聚类"，而是"具体分子的浓度分布"）

**真实优势**
- 从"像素分类"升级为"分子浓度空间映射"——信息量质的飞跃
- 遥感算法已在合成数据和真实数据上经过 20 年验证
- 对肿瘤异质性分析有直接临床价值（同一肿瘤区域内不同代谢表型的空间分布）

**技术可行性评分**：8/10
**创新潜力评分**：9/10（遥感→MSI 的系统性迁移在文献中几乎为零）
**问题清晰度评分**：8/10

**发表潜力**：**Nature Methods**（跨领域算法迁移 + 方法学空白）

**预计投入**：4-5 个月

**前置依赖**：imzML 格式处理基础设施

#### 方向 D：图信号处理（GSP）用于通路感知统计检验

**解决什么问题**

代谢组学差异分析的标准方法是：对每个代谢物做 t 检验/Wilcoxon 检验 → FDR 校正 → 差异代谢物列表 → 通路富集分析（ORA/GSEA）。

这个流程有一个根本性的逻辑缺陷：**先把通路结构丢掉（逐代谢物检验），再试图把它找回来（通路富集）**。代谢物不是独立的——它们通过代谢反应相互连接，一个代谢物的浓度变化会沿通路级联影响下游代谢物。逐代谢物检验假设独立性，这与生化现实矛盾。

**为什么 GSP 是正确的解法**

图信号处理直接在代谢网络上定义"信号"和"变换"：
- 代谢物浓度变化 = 定义在代谢网络图上的信号
- 图拉普拉斯的特征分解 = 代谢网络的"频域"
- 低频成分 = 在网络上平滑变化的系统性代谢改变（通路层面的协调变化）
- 高频成分 = 在网络上不连续的局部异常（可能是测量噪声或真的独立变化）

这不是"把 GSP 方法搬到代谢组学"那么简单——它解决的是一个明确的统计学问题：**如何做通路感知的差异检验**，避免"先丢弃结构、再找回结构"的逻辑矛盾。

**具体技术路线**

1. 从 KEGG/Reactome 构建代谢物网络图（节点=代谢物，边=反应，边权=反应方向性/酶活性）
2. 计算图拉普拉斯矩阵 L，做特征分解：L = UΛU^T
3. 将代谢物浓度差异向量（case - control）投影到图傅里叶基 U
4. 检验：低频分量是否显著 → 系统性通路变化；高频分量是否显著 → 局部异常
5. 输出：不是"差异代谢物列表"，而是"差异通路子网 + 在该子网中的传播方向"

**真实优势**
- 统计功效更高：通路层面的信号比单个代谢物更稳定，减少多重检验惩罚
- 生物学可解释性更强：输出是"通路 X 的代谢流从 A 向 B 方向移动"，不是"代谢物 42 和代谢物 107 上调了"
- 现有工具空白：代谢组学中没有任何工具做"网络层面的统计检验"（FlowGAT 做的是 GNN 预测，不是 GSP 统计检验）

**技术可行性评分**：7/10（图信号处理理论成熟，但需要仔细构建代谢网络图的权重和拓扑）
**创新潜力评分**：9/10
**问题清晰度评分**：9/10（逐代谢物检验的局限性是公认问题，通路富集分析的后处理本质也被广泛质疑）

**发表潜力**：**Nature Communications** 或 **Nature Methods**

**预计投入**：4-5 个月

#### 方向 E：跨引擎标准化评估框架（首篇论文，最高执行优先级）

> 注：虽然算法创新性不是最高，但**执行优先级最高**，因为它是首篇论文、所有后续工作的基础。

**解决什么问题**

代谢组学缺少"GATK Best Practices"式的标准化基准。4 个引擎处理同一数据，特征重叠仅 8%，但研究者没有客观依据选择引擎。

**为什么这不仅仅是工程工作**

表面看是"跑一遍各引擎然后比较"，实际上需要回答一个深层方法学问题：**引擎间不一致的根源在哪里？** 是峰检测算法差异？是参数默认值差异？是特征对齐算法差异？还是底层数据结构（centroid vs profile）的差异？

MetaboFlow 作为唯一能并行运行所有引擎的平台，能做到的不只是"比较结果"，而是"逐步骤归因不一致性"——这是任何单引擎工具无法完成的。

**具体技术路线**

1. 选取 ≥5 个公开数据集（MTBLS），覆盖三种仪器、三种基质、两种色谱模式
2. 4 个引擎（XCMS、MZmine 4、pyOpenMS、asari）以同参数策略并行处理
3. **逐步骤分析**（不只是比最终结果）：
   - 峰检测后的特征列表对比（还没做对齐和归一化之前）
   - 对齐后的对比（区分"峰检测差异"和"对齐差异"）
   - 归一化后的对比
   - 最终差异分析结果的对比
4. 统计建模：哪些因素（仪器、基质、样本量、信噪比）预测引擎间一致性
5. 输出引擎选择决策树

**技术可行性评分**：9/10
**创新潜力评分**：8/10（代谢组学首个全流程逐步骤基准评估）
**问题清晰度评分**：10/10

#### 方向 F：对比学习多平台代谢组数据对齐

**解决什么问题**

同一代谢物在 LC-MS 和 GC-MS 上产生完全不同的谱图（不同的碎裂机制、不同的 m/z 范围）。目前跨平台数据整合只能通过保留时间预测 + 精确质量匹配，准确率很低。这直接导致：
- LC-MS 和 GC-MS 数据无法联合分析
- 不同实验室（不同平台）的结果无法直接比较
- 大规模队列研究（不同中心用不同仪器）的数据整合极其困难

**为什么对比学习是正确的解法**

对比学习的核心是"学习不变表征"——同一实体在不同视角下的表征应该相近。这恰好是跨平台代谢物匹配的本质需求：
- 正样本对：同一代谢物的 LC-MS 谱图 和 GC-MS 谱图
- 负样本对：不同代谢物的谱图
- 学到的嵌入空间中，代谢物身份（而非平台差异）决定距离

CMSSP（Analytical Chemistry 2024）已证明对比学习可以对齐谱图和分子结构，但**LC-MS + GC-MS 的跨平台对比预训练**尚未被探索。

**具体技术路线**

1. 收集训练数据：GNPS 中同一代谢物的 LC-MS/MS 和 GC-MS 谱图对（+分子结构三模态）
2. SimCLR/MoCo 框架训练跨平台嵌入
3. 下游应用：
   - 跨平台代谢物匹配（解决 LC-MS/GC-MS 结果整合）
   - 跨实验室数据对齐（不同中心数据的统一分析）
   - "平台无关"的代谢物表征（存入数据库供未来检索）

**技术可行性评分**：8/10
**创新潜力评分**：8/10
**问题清晰度评分**：8/10

**发表潜力**：**Nature Communications**

**预计投入**：4-6 个月（需 GPU 资源）

#### 方向 G：Pseudotime 代谢动力学轨迹推断

**解决什么问题**

代谢组学时间序列实验（药物干预后 0h/6h/12h/24h 采样）的标准分析是"每个时间点独立做差异分析"。这完全忽略了时间轴上的连续性——代谢物浓度变化是一个动态过程，不是离散快照。

**为什么 Pseudotime 不只是"借用单细胞方法"**

在单细胞组学中，Pseudotime 解决的是"细胞没有时间标签，需要从数据中推断发育轨迹"。在代谢组学中，样本**有**时间标签，但 Pseudotime 框架仍然有用，因为：
- 不同个体的代谢反应速度不同（同一时间点的样本可能处于不同的代谢状态）
- 代谢轨迹可能有分支（同一干预导致不同个体走向不同的代谢结局）
- Pseudotime 可以将"10 个时间点 × 50 个个体"的数据整合为一条连续的代谢轨迹，而不是 10 个独立的差异分析

2025 年 bioRxiv 已有单细胞脂质组学的 proof-of-concept，但**批量代谢组学的系统实现是空白**。

**具体技术路线**

1. 将多时间点代谢组样本投影到 UMAP/PCA 空间
2. 图最小生成树推断代谢轨迹，允许分支
3. 沿轨迹计算代谢物变化趋势，识别轨迹关键节点（"代谢状态转变点"）
4. 验证：用已知时序数据集（脂肪细胞分化、药物代谢动力学）测试

**技术可行性评分**：8/10
**创新潜力评分**：9/10
**问题清晰度评分**：7/10（时间序列代谢组学用户群体相对小）

**发表潜力**：**Nature Communications** 或 **PLOS Computational Biology**

**预计投入**：3-4 个月

#### 方向 H：HHT/EMD 用于色谱基线漂移自适应去除

**解决什么问题**

LC-MS 色谱信号的基线不是平的——会有缓慢漂移（仪器温度变化、流动相组成梯度、色谱柱老化）。所有后续分析（峰检测、定量）的准确性都依赖于正确的基线去除。

**现有方法的问题**

当前方法都需要**假设基线的形状**：
- 多项式拟合：假设基线是 N 次多项式（N 由用户选择）
- LOESS 回归：假设基线是局部平滑的
- 小波方法：假设基线对应特定频率范围

这些假设在复杂基质样品（血浆、组织提取物）中经常不成立——基线可能有突变、多段不同趋势、化学干扰导致的局部凸起。

**为什么 EMD 是正确的解法**

EMD（经验模态分解）是**唯一不需要预设基函数**的信号分解方法。它通过迭代包络平均自适应地将信号分解为 IMF（内在模式函数），每个 IMF 有自己的特征频率：
- 最低频 IMF = 基线漂移
- 中频 IMF = 真实色谱峰
- 最高频 IMF = 仪器噪声

关键：EMD 不需要用户告诉它"基线长什么样"——它自己从数据中发现。

**代谢组学领域 HHT/EMD 应用文献近乎空白**，而在地震信号、脑电信号、金融数据中已有大量成功应用。

**具体技术路线**

1. 用 EEMD（集成 EMD，减少模态混叠）分解每条 EIC
2. 自动识别 IMF 分类：Hurst 指数区分噪声 IMF（H<0.5）、峰 IMF（H~0.5-0.8）、基线 IMF（H>0.8）
3. 将基线 IMF 从原始信号中减去 → 得到基线校正后的信号
4. 与多项式/小波/LOESS 基线校正方法在不同样品类型上系统对比

**技术可行性评分**：9/10（`PyEMD` 库成熟，代码实现简单）
**创新潜力评分**：8/10
**问题清晰度评分**：8/10

**发表潜力**：**Analytical Chemistry**

**预计投入**：2-3 个月（快速出成果的方向）

#### 方向 I：混合裂解谱深度学习解卷积

**解决什么问题**

DDA（Data-Dependent Acquisition）模式下，MS2 采集窗口有一定宽度（通常 1-2 Da）。如果两个不同前体离子的 m/z 差异在窗口内，它们的裂解碎片会混合在同一张 MS2 谱图中。这张混合谱图拿去做数据库匹配，匹配结果是错的——因为谱图是两个分子的叠加。

DIA（Data-Independent Acquisition）数据中这个问题更严重：每个宽窗口（10-50 Da）内可能有数十个前体离子同时碎裂。

**为什么借鉴音频源分离**

混合裂解谱的数学模型与鸡尾酒会问题完全对应：
- 鸡尾酒会：麦克风录到的 = 说话人 A 的声音 + 说话人 B 的声音 + 环境噪声
- 混合 MS2：检测器记录的 = 分子 A 的碎片谱 + 分子 B 的碎片谱 + 化学噪声

深度学习源分离方法（Conv-TasNet、DPRNN）在 2019-2023 年已将音频分离质量推到接近人类水平。这些模型的输入输出结构可以直接迁移到谱图分离：
- 输入：混合谱图（m/z × intensity）
- 输出：K 个分离后的纯谱图

**不只是类比——有具体的迁移路径**

1. **编码器**：将 m/z 轴离散化为 bins（类似音频的 STFT），得到"频谱图"
2. **分离网络**：Conv-TasNet 的 TCN（时序卷积网络）替换为 1D 卷积（m/z 轴上的卷积）
3. **解码器**：输出 K 个分离后的谱图
4. **训练数据**：从 MoNA/GNPS 采样单谱图 → 随机线性叠加 → 合成混合谱图（ground truth 已知）

**真实优势**
- DIA 数据的利用效率大幅提升（当前很多 DIA 数据因为解卷积困难而未被充分利用）
- 与现有方法（DIA-NN 的谱图库匹配、SWATH-MS 的离子色谱提取）互补

**技术可行性评分**：7/10（需要大量合成训练数据；模型泛化性待验证）
**创新潜力评分**：9/10
**问题清晰度评分**：8/10

**发表潜力**：**Analytical Chemistry** 或 **Journal of Proteome Research**

**预计投入**：4-5 个月

#### 方向 J：最优传输谱图相似度（OT-based Spectral Similarity）

**解决什么问题**

余弦相似度将质谱视为高维向量，计算向量夹角。但它有一个根本的"几何盲区"：m/z = 100.0000 的峰和 m/z = 100.0050 的峰（5 ppm 偏差，同一个离子的测量误差范围内）对余弦相似度的贡献为零——因为它们在不同维度上。

**为什么 OT 是正确的解法**

Wasserstein 距离（最优传输距离）将谱图视为 m/z 轴上的概率分布，计算"将一个分布搬运到另一个分布所需的最小代价"。这天然考虑了 m/z 轴上的邻近关系——100.0000 和 100.0050 的峰只需要很小的搬运代价。

**深度可行性分析**

- 正面：WABI 2018 已有概念验证；POT 库的 Sinkhorn 算法足够快
- **挑战**：FlashEntropy 已将余弦相似度加速到微秒级，OT 的速度（毫秒级）在大型谱库检索中可能是瓶颈
- **解法**：先用 FlashEntropy 做粗筛（Top-1000），再用 OT 做精排序（两阶段检索）
- **关键实验**：需要在同分异构体数据集上证明 OT 比余弦有统计显著的优势，否则速度劣势无法被接受

**技术可行性评分**：9/10
**创新潜力评分**：7/10（概念不新，价值在于系统性评估）
**问题清晰度评分**：8/10

**发表潜力**：**Analytical Chemistry**

**预计投入**：3-4 个月

---

#### 方向 K：保形预测（Conformal Prediction）用于注释不确定性量化

**解决什么问题**

现有注释工具输出"置信度分数"（如 SIRIUS 的 zodiac score），但这个分数没有统计学保证——0.95 的分数不意味着 95% 的概率是正确的。Conformal Prediction 提供有统计学保证的预测集："以 α=0.05 的错误率保证，真实答案在这个候选集内"。

**深度可行性分析**

- **实现成本极低**：核心代码 <200 行，只需要校准集 + 非一致性分数计算
- **真实价值**：将注释从"可能是 X（分数 0.87）"升级为"以 95% 概率保证，真实答案在 {X, Y, Z} 中"
- **局限**：如果候选集太大（包含 50 个分子），用户体验没有改善。关键指标是"在给定覆盖率下，候选集的平均大小"
- 2024-2025 年已有 2-3 篇相关工作（IJMS 2024, PLOS 2025），但系统性覆盖全注释流程（分子式→结构→通路）的框架是空白

**技术可行性评分**：9/10
**创新潜力评分**：7/10
**问题清晰度评分**：8/10

**发表潜力**：**Analytical Chemistry** 或 **Bioinformatics**

**预计投入**：2-3 个月

#### 方向 L：Target-Decoy FDR 统一框架

**解决什么问题**

蛋白质组学有成熟的 FDR 控制（target-decoy 策略已成为标准步骤），代谢组学几乎没有。当前代谢组学工具报告的"Level 2 注释"中，假阳性率是未知的。

**深度可行性分析**

- **核心挑战**：代谢组学的 decoy 生成比蛋白质组学困难。蛋白质组学可以反转氨基酸序列生成 decoy，但代谢物没有"序列"。现有 4 种代谢组 decoy 策略（极性翻转、镜像、采样、次级排名）各有局限
- **具体实现**：在 MetaboFlow 注释流程中同时运行 target 和 decoy 检索，输出 FDR 估计值
- **价值判断**：这不是算法创新，而是**最佳实践建立**——但在代谢组学中同样急需

**技术可行性评分**：8/10
**创新潜力评分**：7/10（2024 年已有零散工作）
**问题清晰度评分**：9/10

**发表潜力**：**Analytical Chemistry**

**预计投入**：2-3 个月

#### 方向 M：天文 PSF 反卷积 → 峰形恢复

**解决什么问题**

质谱仪器测量的峰形不是真实的分子信号，而是真实信号与仪器响应函数（类似天文学的 PSF）的卷积。两个 m/z 接近的分子，它们的峰可能在仪器响应函数的影响下完全重叠。

**深度可行性分析**

- **核心假设**：仪器 PSF 是已知的或可建模的。对于 Orbitrap（理论峰形为 Lorentzian）和 QTOF（理论峰形为 Gaussian），这个假设基本成立。但实际 PSF 随 m/z 变化，需要校准
- **具体实现**：Richardson-Lucy 迭代（天文学标准方法），输入是观测谱图和 PSF 模型，输出是去卷积后的"真实"谱图
- **真实价值点**：在高分辨质谱上价值有限（Orbitrap 60K 分辨率已经够用），但在**低分辨质谱（三重四极杆、离子阱）**上可能有显著价值
- **前置需求**：需要 PSF 校准实验数据（不同 m/z 处的峰形测量），增加了执行难度

**技术可行性评分**：7/10
**创新潜力评分**：8/10（在低分辨质谱上的应用有明确空白）
**问题清晰度评分**：7/10（高分辨质谱用户可能不需要）

**发表潜力**：**Analytical Chemistry**

**预计投入**：4-5 个月

### 2.4 已删除方向及理由

以下 3 个方向从 V1.0 中删除，理由如下：

#### ~~方向 N（原）：金融 GARCH → 批次漂移建模~~

**删除理由**：
1. **类比表面化**：GARCH 解决的是"波动率聚集"（volatility clustering）——高波动期后更可能出现高波动。但 LC-MS 的批次漂移不具备这个统计特征，批次漂移是缓慢趋势性的（温度变化、色谱柱老化），不是波动率聚集
2. **现有方法已够用**：QC-RLSC（QC-Robust LOESS Signal Correction）和 ComBat 已经是成熟的批次效应校正方法，GARCH 不解决它们不能解决的问题
3. **问题-方法匹配度低**：硬把金融时间序列模型套到色谱信号上，缺乏第一性原理支撑

#### ~~方向 O（原）：Portfolio 优化 → 生物标志物 Panel 选择~~

**删除理由**：
1. **类比成立但不够创新**：将 Markowitz 均值-方差框架用于特征选择，本质就是"考虑了特征间相关性的正则化选择"——L1/L2 正则化 + 相关性约束已经能做到
2. **Elastic Net + 相关性滤波** 在实际效果上很难被 Portfolio 框架超越，因为底层数学模型是等价的
3. **发表空间窄**：Bioinformatics 审稿人大概率会问"这比 Elastic Net 好在哪"，而我们很难给出令人信服的答案

#### ~~方向 P（原）：参数自动推荐系统~~

**删除理由**：
1. **纯工程问题**：这是产品功能，不是算法创新。"基于历史数据推荐参数"用简单的 lookup table 或 random forest 就能做到
2. **依赖数据积累**：需要平台有足够的历史数据才有意义，不适合作为早期科研方向
3. **已有竞品**：IPO（Isotopologue Parameter Optimization）和 AutoTuner 已经解决了参数优化问题，推荐系统只是在此基础上加了一层缓存
4. **发表上限低**：方法学短文级别，不值得专门投入

### 4.2 合成基准数据集设计

合成数据集的核心价值：**ground truth 完全可控**，可以精确评估假阳性率和假阴性率。

**设计方案**

1. **混合标准品数据集（MetaboMix Benchmark）**
   - 覆盖 50-100 个代谢物标准品（来自 NIST SRM、Sigma MetaSep 等商业混合物）
   - 两组设计：组 A 包含标准品混合，组 B 额外加入"已知差异代谢物"（浓度增加 2/5/10 倍）
   - 三种仪器分别采集：Thermo Orbitrap Q Exactive、Waters QTOF、Bruker timsTOF
   - Ground truth：已知每个代谢物的精确浓度，可直接计算每个引擎的真阳性率/假阳性率

2. **计算机模拟数据集**
   - 使用 RawTools 或自实现工具，在已知谱图上叠加不同水平的噪声、基线漂移、保留时间偏移
   - 系统性评估各引擎在不同 SNR 条件下的峰检测性能
   - **GID 专用模拟数据**：已知分子列表 → 离子化物理模型 → 合成原始数据 → GID 能否正确恢复分子列表

### 4.3 跨引擎基准测试协议

**标准化执行协议（MetaboFlow Benchmark Protocol V1）**

```
步骤 1：数据准备
  - 原始文件用 MSConvert 3.x.x 统一转换为 mzML（centroid 模式）
  - 记录转换参数和 MSConvert 版本号
  - 计算原始文件 MD5 校验和

步骤 2：参数策略
  - 每个引擎使用"MetaboFlow Best Practices"默认参数（基于文献推荐）
  - 同时测试"用户自定义优化参数"（用 IPO/AutoTuner 自动优化）
  - 参数完整记录到 MetaboData.uns['processing_params']

步骤 3：并行执行
  - 同一数据集，4 个引擎并行运行（不同 Docker 容器，确保环境隔离）
  - 每个引擎运行 3 次（检测可重复性）
  - 记录运行时间和内存峰值

步骤 4：结果标准化
  - 每个引擎输出转为 MetaboData 统一格式
  - 按 m/z 容差（±5 ppm）和 RT 容差（±0.1 min）跨引擎匹配特征

步骤 5：评估指标计算
  - 特征数量、重叠率（Venn 图）
  - 注释率（用 matchms + MoNA 统一注释）
  - CV 分布（QC 样品的特征级变异系数）
  - 下游差异分析的差异代谢物重叠率
```

## 6. 知识产权策略

### 6.1 开源 vs 专利的边界

**开源范围（MIT 协议）**

以下内容全部开源：
- MetaboFlow 核心引擎聚合框架
- MetaboData 中间格式定义和 SDK
- 所有引擎适配器
- 跨引擎基准测试框架和评估代码
- 所有算法创新的研究实现（包括 GID 和 COCA）
- 标准图表模板库

**理由**：
1. 开源是发表 NM/NC 级论文的前置条件
2. 社区效应 > 专利保护的商业价值
3. MetaboFlow 的竞争壁垒在于**平台效应和数据积累**（尤其是 COCA 依赖的跨样本共检测数据），而非代码保密

**商业化范围（不开源）**

- Web SaaS 前端界面
- 企业私有部署服务和 SLA 合同
- COCA 的共检测数据库（平台数据资产）

### 6.2 先发表还是先申请专利

**结论：先发表**

判断依据：代谢组学算法的商业价值在于平台体验，不在于算法本身。bioRxiv 预印本提供时间戳保护，GitHub commit 历史证明开发时间线。
