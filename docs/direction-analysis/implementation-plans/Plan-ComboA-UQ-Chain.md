# 组合A — 端到端不确定性传播链：科研实施计划

---

## 一、研究背景

### 1.1 代谢组学注释问题的领域现状

代谢组学通过非靶向质谱技术在单次实验中同时检测数千个代谢特征，已成为系统生物学的核心工具。然而，代谢物注释至今仍是这一领域的技术瓶颈。与蛋白质组学中肽段序列数据库检索的高确定性不同，代谢物注释面临根本性的歧义性：相似质量的候选化合物可达数十甚至数百个，MS/MS碎片匹配存在固有的不完全性，同分异构体难以区分。

国际代谢组学协会（MSI）的置信度分级标准将注释分为四个层次：Level 1为经标准品验证的确定注释，Level 2为基于谱图库匹配的推定注释，Level 3为基于化合物类的推断注释，Level 4为未知化合物。在典型的非靶向代谢组学研究中，能达到Level 1的特征通常不足20%，而大量被纳入下游分析的特征实际上处于Level 2甚至Level 3的注释状态。

现有注释工具代表了该领域的技术前沿。MetFrag（Ruttkies et al., 2016）通过对候选分子的理论碎裂模式进行评分来筛选候选注释，但输出的是单一最优候选而非概率分布。SIRIUS（Dührkop et al., 2019）引入了分子指纹和化学结构预测，并通过CANOPUS进行化合物类推断，是目前自动化程度最高的注释工具之一，但同样以确定性注释作为最终输出。GNPS（Wang et al., 2016）提供了社区维护的谱图库和网络分析框架，在天然产物研究中被广泛使用。MassID（假设竞争工具）和BAUM等框架尝试在部分环节引入不确定性量化，但均未实现从注释到差异分析到通路分析的全链路不确定性传播。

上述工具的共同局限在于：它们将注释不确定性视为需要在流程入口处消除的噪声，而非贯穿整个分析流程的统计信号。这一设计决策在注释步骤产出硬标签时将不确定性截断，使下游分析失去了一个重要的信息维度。

### 1.2 Conformal Prediction在科学应用中的发展

Conformal Prediction（共形预测，CP）是一种统计学习框架，由Venn和Gammerman于2005年提出，Angelopoulos和Bates于2023年发表的综述对其理论基础和应用进行了系统梳理。CP的核心特性是在无分布假设的前提下提供有效覆盖率保证：对于目标覆盖率1-ε（如90%），CP保证真实标签以至少1-ε的概率落入预测集。这一保证仅依赖于校准集与测试集之间的可交换性（exchangeability），不需要同分布假设。

CP框架近年来在科学计算领域获得了越来越多的应用。在蛋白质结构预测中，CP被用于为AlphaFold2的预测结构生成置信区间（Luo et al., 2023）。在基因组变异注释中，CP被用于控制致病性预测的假阳性率。在药物发现中，CP被用于为分子性质预测提供可靠的不确定性估计。

然而，CP在代谢组学注释中的应用至今几乎空白。这一空白的存在有其合理性：代谢组学注释问题的多维性（质量偏差、谱图相似度、保留时间、碰撞截面积多个证据来源）给CP的conformity score设计带来了相当的复杂性；校准集与测试集之间的化学空间差异（标准品 vs 真实样本中的代谢物）对exchangeability假设构成挑战；同分异构体问题使得"正确注释"本身难以明确定义。本研究直接面对这些挑战，提出代谢组学专用的CP实施方案。

### 1.3 差异分析中"等权重假设"的问题

当前代谢组学差异分析的标准流程是：将注释后的代谢特征矩阵输入limma（Smyth, 2004）或类似工具，每个特征被等同对待，差异分析结果（logFC、p值）与注释置信度无关。这一"等权重假设"在统计方法层面过于简单，在生物学层面则引入了系统性偏差。

等权重假设的问题可从两个角度分析。其一，高置信注释特征（如氨基酸，谱图匹配分>0.9）和低置信注释特征（如脂质异构体，谱图匹配分0.3-0.5）在差异分析中获得相同的统计权重，但后者的差异信号高度可能反映的是注释错误而非真实生物学差异。其二，在多重检验校正中，等权重假设隐含了所有特征的先验差异概率相同，但显然高置信注释的特征更可能是真实的生物学活性代谢物，其先验H₁概率应更高。

Independent Hypothesis Weighting（IHW，Ignatiadis et al., 2016）为解决这一问题提供了统计学框架：通过与p值独立的协变量对假设进行分层，不同层使用不同的α阈值，在控制全局FDR的前提下提高统计功效。然而，IHW在代谢组学中的应用迄今主要使用特征强度或检测率作为协变量，而非注释置信度——后者是更直接、更具生物学意义的协变量选择。

### 1.4 通路分析中"硬标签"的局限性

通路富集分析是代谢组学研究从特征列表到生物学解释的最后一跳。过设分析（ORA）和基因集富集分析（GSEA）的代谢物版本（如MetaboAnalystR实现）都以确定性的代谢物注释作为输入：将差异代谢物列表与通路数据库进行比对，通过超几何检验或置换检验评估通路富集显著性。

这一硬标签输入机制的局限性在同分异构体问题上最为突出。当一个质谱特征的预测集包含分属不同通路的同分异构体（如亮氨酸和异亮氨酸分属mTOR信号通路和支链氨基酸代谢通路）时，传统方法被迫做出一个武断的选择：取top-1注释（引入系统偏差）或丢弃该特征（损失统计功效）。任何一个选择都是信息的损失。

更广泛地看，传统通路分析将注释不确定性视为"分析前的噪声"而不是"分析中的信号"，导致通路后验推断无法反映其依赖的注释证据质量。一个基于低置信注释的"显著"通路和一个基于高置信注释的"显著"通路，在报告中获得相同的处理，但其生物学可靠性存在本质差异。

### 1.5 文献脉络与研究空白

综合上述分析，现有方法体系在三个关键节点存在系统性缺陷：注释阶段产出硬标签截断不确定性、差异分析等权重丢失置信度信号、通路分析硬标签输入无法传播注释歧义。这三个断点共同导致了一个可观察到的现象：不同实验室对同一数据集运行相似流程，通路分析结论的重现率出人意料地低（Spicer et al., 2017）。注释不确定性的未量化传播是这一可重现性危机的重要但尚未被充分认识的贡献因素。

本研究的核心主张是：注释不确定性在代谢组学全流程中是一阶效应，而非可忽略的系统噪声。端到端不确定性传播不仅在理论上更严格，在实践中也能产生实质性不同的生物学结论。

## 二、科学问题

本研究明确回答以下三个层次的科学问题：

**核心问题一：注释不确定性量化是否是下游分析结论的一阶效应？**

即：在控制其他变量的前提下，使用概率注释集（CP prediction set）与硬标签注释相比，差异代谢物列表和通路富集结论是否存在统计显著和生物学意义的差异？差异的量级是否足够大，以至于影响实质性的生物学解读？

**核心问题二：端到端传播是否优于分段处理？**

即：三阶段UQ传播（Phase 1注释→Phase 2差异→Phase 3通路的全链路传播）相比于仅在单一阶段引入不确定性量化，是否在跨队列可重复性和通路推断稳定性上具有可量化的优势？是否存在非线性协同效应，使得整体大于部分之和？

**核心问题三：不同置信度层次的注释对通路推断贡献的异质性有多大？**

即：哪些通路的推断对注释不确定性高度敏感（不稳定通路），哪些通路的推断在注释假设变化下保持稳定（稳定通路）？这种异质性是否具有系统性规律（如特定代谢物类别、特定通路结构特征）？

## 三、研究意义

### 3.1 方法学范式转变

当前代谢组学的方法论范式建立在确定性注释的基础上，将注释问题视为分析流程的预处理步骤而非贯穿全程的统计推断过程。本研究提出的框架代表了一种范式转变：将代谢组学注释重新定义为一个产出概率分布而非点估计的推断问题，并要求这一概率分布在后续所有分析步骤中得到严格传播。

这一转变的意义不局限于技术层面。它要求研究者在报告结论时对注释证据质量作出明确陈述，使"基于低置信注释的通路发现"和"基于高置信注释的通路发现"在科学报告中获得不同的处理——这是对科学严谨性的直接推进。

### 3.2 对可重复性危机的回应

代谢组学领域的可重复性问题近年来受到广泛关注。Spicer等（2017）对多个代谢组学研究的重分析显示，即使使用相同数据和相似方法，通路分析结论的跨实验室重现率常常低于预期。本研究通过以下机制直接回应这一问题：首先，CP框架提供了数学上可证明的覆盖率保证，使注释置信度在不同实验室、不同数据集之间具有可比性；其次，Monte Carlo稳定性分析将通路结论分为依赖特定注释假设的"不稳定"结论和对注释变化鲁棒的"稳定"结论，为研究者提供了评估结论可靠性的量化工具；第三，通过跨队列验证证明，稳定通路（Tier 1）在独立队列中的一致性显著高于不稳定通路（Tier 3），为"什么样的通路发现值得相信"提供了实证标准。

### 3.3 对同分异构体问题的系统性回应

同分异构体难以区分是代谢组学注释歧义的核心来源之一，尤其在脂质组学中极为突出。本框架提供了一个系统性的处理路径：不试图强行区分同分异构体，而是通过概率分配将同分异构体的差异信号按各候选的注释概率分配给相应的代谢通路。这一处理不是技巧性的折中，而是从统计学原理出发的正确做法——在证据不足以区分时，诚实报告不确定性并将其传播给结论，而非人为制造确定性。

## 四、创新点

### 4.1 Phase 1创新：四维conformity score

**创新内容：** 提出代谢组学专用的四维联合conformity score，将MS/MS谱图余弦相似度、m/z质量误差高斯核、保留时间匹配项和碰撞截面积（CCS）匹配项整合为单一的、可校准的conformity score。

**为什么这是新的：** 现有的代谢物注释打分系统（MetFrag评分、SIRIUS后验概率）虽然也整合多维证据，但其输出不满足CP所需的conformity score特性——即在校准集上的分位数可以直接用于测试集的覆盖率保证。本研究的四维conformity score专门为CP校准而设计，使四维证据的联合整合产生具有统计保证的概率预测集，而非单纯的候选排名。

**为什么这样设计：** m/z和CCS项使用高斯核而非硬阈值，保留了距离信息（误差越小分越高）而非二值化判断；谱图相似度使用modified cosine similarity（考虑中性丢失偏移）而非简单内积，对低质量MS2光谱更鲁棒；自适应权重通过在校准集上的逻辑回归学习最优组合，将四维证据整合从人工设置转变为数据驱动问题。这一设计允许框架适应不同仪器、不同化合物类别的证据可靠性差异。

### 4.2 Phase 1创新：CCS贝叶斯后验更新

**创新内容：** 将实测CCS（来自离子迁移率质谱数据）作为CP之后的后处理步骤进行贝叶斯更新，而非作为并行证据源并入conformity score。

**为什么这是新的：** 现有工具（MetFrag、SIRIUS、AllCCS2）通常将CCS作为另一个打分维度，与MS2和m/z并列整合。本研究将CCS的处理分离出来，置于CP之后。其理由是：对于大多数LC-MS数据，CCS数据不可用（非离子迁移率仪器）；将CCS纳入conformity score会使无CCS数据时的校准集构建复杂化；而贝叶斯后验更新允许CCS证据在有实测值时无缝改善CP prediction set，在无实测值时自动退化为弱信息先验而不引入偏差。这一两步结构（CP生成初始预测集，贝叶斯更新进一步精化）在代谢组学注释中是首次提出。

**为什么这样设计：** 贝叶斯框架下，CP给出的边际概率作为先验，CCS误差模型（高斯误差，相对标准差2.5%，来自AllCCS2验证集）提供似然，后验归一化后作为更新的候选概率。这一设计保持了概率归一化，避免了将CCS惩罚与CP阈值混合时的定标问题，同时为不同精度的CCS测量（DTIMS vs TIMS-TOF）提供了统一的处理框架。

### 4.3 Phase 1创新：化学骨架分层的校准集划分策略

**创新内容：** 针对代谢组学CP应用中exchangeability假设难以满足的问题，提出基于Bemis-Murcko化学骨架的分层划分策略（Scaffold Split变体），并结合ClassyFire化合物类的代表性约束，使校准集和测试集的化学多样性更接近真实的exchangeability条件。

**为什么这样设计：** 随机划分在代谢组学CP中会引入系统性乐观偏差——结构相似的化合物天然具有相近的conformity score，若同一化学类在校准集和测试集中都有大量代表，coverage在测试集上会虚高。Scaffold Split通过在化学骨架层面隔离校准集和测试集，更接近实际应用中标准品（校准集）与真实样本代谢物（测试集）之间的化学差异。这种保守划分下，实际coverage会略高于nominal coverage——在科学应用中，覆盖率超出比不足更可接受。

### 4.4 Phase 2创新：注释概率作为limma precision weight及β的数据驱动选择

**创新内容：** 将注释置信度概率通过指数变换（权重=概率^β）注入limma线性模型作为precision weight，并提出数据驱动的β选择方法（在spike-in标准品实验上通过最大化AUPR确定最优β）。

**为什么这是新的：** limma的precision weight机制本身已有定义，用于处理样本质量差异。将注释置信度作为特征级precision weight注入，在概念上是将注释的认识不确定性（epistemic uncertainty）纳入差异分析的统计模型。IHW此前以特征强度或检测率作为协变量，但注释置信度作为IHW协变量选择是本研究的新应用——注释置信度与差异p值在统计上独立（前者来自谱图匹配，后者来自样本间强度分布），满足IHW的核心假设，且携带比特征强度更直接的生物学先验信息。

**β的独创设计意义：** β=0退化为传统等权重方法，β=1是期望无偏估计，β>1对高置信特征进行超线性强化。β的连续调节将"要多激进地使用注释置信度信息"转化为一个可量化、可优化的参数问题，而非研究者的主观判断。通过spike-in实验的数据驱动选择，为这一关键参数的确定提供了可重现的方法论基础。

### 4.5 Phase 2创新：同分异构体差异信号分配公式

**创新内容：** 对prediction set中包含同分异构体候选的模糊特征节点，提出差异信号按候选概率加权分配的数学公式体系，包括logFC的线性分配和t统计量的平方根比例缩放分配，并设计了p值向通路层面传递的概率加权公式。

**为什么这样设计：** logFC的线性分配（logFC_attributed = logFC × q_k，q_k为同分异构体子集内的归一化概率）具有直接的解释性：若一个特征60%可能是亮氨酸，则该特征对亮氨酸相关通路的logFC贡献应是其观测logFC的60%。t统计量采用平方根比例（t_attributed = t × √q_k）而非线性比例，是因为t统计量的信噪比在标准差尺度而非方差尺度上线性，q_k描述的是概率（方差比例），因此在标准差尺度上的等效信息贡献是√q_k。这一处理使得信号分配后的t统计量分布在统计检验框架内保持合理的解释性。

### 4.6 Phase 3创新：三层嵌套Monte Carlo扰动体系

**创新内容：** 提出三层嵌套的Monte Carlo扰动设计：层1（注释重采样，700次）通过从预测集按概率分布重采样注释模拟注释决策的随机性；层2（β扰动，200次）评估差异分析权重参数的敏感性；层3（校准集子集扰动，100次）验证CP校准的稳定性。三层扰动设计使通路结论稳定性的来源（注释歧义、权重参数、校准数据）得以解耦量化。

**为什么这是新的：** BAUM使用jackknife敏感性分析，但不重采样注释集本身，因此无法量化注释决策对通路结论的直接影响。本研究的层1扰动直接模拟"如果注释按概率随机实现，通路结论的稳定性如何"，是更本质的sensitivity测试。三层分离允许研究者区分"通路结论不稳定是因为注释歧义"（层1主导）和"通路结论不稳定是因为分析参数选择"（层2主导）这两类性质不同的不确定性来源。

**Tier分类的设计意义：** Tier 1（>80%扰动下显著）、Tier 2（50-80%）、Tier 3（<50%）的三级分类为研究者提供了报告可靠性的明确语言：Tier 1通路是"注释假设鲁棒的发现"，Tier 3通路是"注释依赖性结论，需独立验证"。这一分类在科学报告中比单纯的p值更诚实地传达了分析的不确定性。

## 五、实验设计

### 5.1 数据集

**校准集：GNPS + HMDB标准品谱图库**

使用GNPS公开谱图库（约15万条标准品MS2谱图，含SMILES/InChI结构信息）和HMDB实验MS2子集构建CP校准集。通过Bemis-Murcko化学骨架分层划分，保留80%用于校准，20%用于测试集验证。校准集覆盖脂质、氨基酸、有机酸、核苷酸、酚类等主要代谢物类别，每类至少30个化合物代表。

**验证数据集一：ST001050（糖尿病队列，189样本）**

来源于Metabolomics Workbench的2型糖尿病研究数据集，包含正常糖耐量（NGT）、糖耐量受损（IGT）和2型糖尿病（T2D）三组受试者的LC-MS正离子模式数据。选择该数据集的理由：样本量在代谢组学研究中具有代表性，疾病组别的代谢差异（糖酵解、支链氨基酸、脂质代谢）在文献中有充分记录，可作为通路分析正向验证的参照。

**验证数据集二：MTBLS214（结直肠癌队列，100样本）**

来源于MetaboLights的结直肠癌与健康对照研究，包含LC-MS/MS正负离子模式数据。选择该数据集的理由：与ST001050疾病类型不同，可用于评估框架的跨疾病通用性；同时作为跨队列可重复性验证的独立数据集。

**验证数据集三：Dunn CAMCAP（299样本）或等效替代**

CAMCAP（Cambridge Cancer Cohort）代谢组学数据，若无法获取则使用MTBLS404（肺癌队列，338样本，MetaboLights完全公开）替代，科学等效性相近。该数据集主要用于跨队列验证，不参与模型训练。

### 5.2 Phase 1验证实验（M3-4月）

**实验P1-A：CP覆盖率校准验证**

目标是证明在标准品测试集上，名义覆盖率（nominal coverage，如90%）与经验覆盖率（empirical coverage）之差在±2%以内，且对八个主要化合物类别均成立。实验变量包括覆盖率水平（70%、80%、90%、95%）和化合物类别（脂质、氨基酸、有机酸、核苷酸等八类），数据来源为NIST标准品的scaffold split测试集。主要输出是覆盖率校准曲线，比较本框架与简单阈值方法和等权重排名方法的校准质量。

**实验P1-B：预测集大小与注释不确定性的关系验证**

目标是验证CP预测集大小确实反映真实不确定性，即高不确定性区域（脂质同分异构体、低质量MS2的化合物）应获得更大的预测集。实验变量包括化合物类别、特征强度和MS2碎片数，使用NIST标准品和ST001050前100个特征的混合数据集。

**实验P1-C：CCS贝叶斯更新效果评估**

目标是量化加入CCS后预测集大小的缩减比例和top-1候选准确率的提升。实验变量包括是否使用CCS和CCS测量噪声水平（σ=1%、2.5%、5%，对应不同仪器精度），数据来源为有CCS实测值的标准品子集（约300-500个化合物，从METLIN CCS数据库获取）。

### 5.3 Phase 2验证实验（M6-8月）

**实验P2-A：β选择实验（核心实验）**

目标是在已知ground truth的条件下确定最优β参数。通过在NIST标准品数据集上构建计算spike-in实验（对已知注释特征的强度矩阵人工添加随机差异，模拟case/control信号），对β取值{0, 0.5, 1, 1.5, 2, 3}运行加权limma，以AUPR（precision-recall曲线下面积）评估各β的性能，以已知spike-in特征作为ground truth标签。预期最优β落在1.0-1.5区间，三个数据集的β-AUPR曲线应具有相近的峰值位置（跨数据集一致性是β参数通用性的关键证据）。

**实验P2-B：FDR控制对比**

目标是验证加权框架不牺牲FDR控制能力，即在提高真阳性发现的同时，实际FDR不超过名义FDR。实验变量包括样本量（50、100、189）和差异倍数（FC=1.5、2、3），基准方法包括等权重limma+BH、等权重limma+qvalue、IHW（无注释权重），主要输出是名义vs实际FDR对比和真阳性数量对比。

**实验P2-C：ST001050真实差异分析**

目标是在真实糖尿病队列上比较本框架与竞争方法的差异代谢物列表。输出Venn图展示本框架、MassID（或可比工具）和传统limma+BH三方法的发现集合，重点分析本框架独有发现中是否含有已知糖尿病生物标志物（正向验证）和通路富集是否更集中于已知糖尿病代谢异常通路（间接验证）。

**实验P2-D：同分异构体信号分配案例研究**

选择至少三对代谢学意义不同的同分异构体对（氨基酸类：亮氨酸/异亮氨酸；甘油磷脂类：PC(36:4)的不同双键位置异构体；鞘脂类：HexCer双键/饱和度同分异构体），展示信号分配前后相关通路后验概率的量化差异。对于亮氨酸/异亮氨酸案例，利用其在多数LC柱上约0.1-0.3分钟的保留时间差异作为RT匹配项的验证基准。

### 5.4 Phase 3验证实验（M10-11月）

**实验P3-A：通路后验vs传统通路富集对比（ST001050）**

目标是直接比较贝叶斯通路后验概率与传统GSEA p值的通路排名差异，量化"排名翻转"通路的数量和生物学特征。散点图（y轴：贝叶斯后验，x轴：传统GSEA -log10(p)）的四个象限分别对应不同的一致性/差异性模式，重点分析离散最远的通路（两方法结论相反的通路）是否具有可解释的注释不确定性来源。

**实验P3-B：Monte Carlo稳定性分类与跨队列一致性验证**

目标是证明Tier分类具有预测效力：Tier 1通路（高稳定性）的跨队列一致性应显著高于Tier 3通路（低稳定性）。具体验证：在ST001050上运行全流程得到Tier分类→在MTBLS214上独立运行→计算Tier 1通路在两数据集通路排名中的Kendall τ vs Tier 3通路的Kendall τ。预期Tier 1通路的跨队列τ>0.6，Tier 3通路的τ<0.3。

**实验P3-C：结直肠癌MTBLS214通路分析及独立验证**

在MTBLS214数据集上运行完整框架，与文献记录的结直肠癌代谢特征（色氨酸代谢、胆碱磷脂代谢等通路）进行正向对比，同时与Dunn CAMCAP（或等效替代数据集）的结果进行跨研究一致性分析。

### 5.5 端到端整合实验（M11-12月）

**实验E2E-A：端到端 vs 分段消融对比**

设计四条分析pipeline：完整三阶段UQ传播（Full）、仅Phase 1+3无Phase 2、仅Phase 2+3无Phase 1、无任何UQ的传统流程（Baseline）。在三个数据集上分别运行，主要评估指标为跨队列通路排名一致性（Kendall τ）和Tier 1稳定通路数量。这一消融实验直接回答"端到端传播优于分段处理"的科学问题。

**实验E2E-B：与MassID、BAUM和传统流程的综合对比**

在所有三个真实数据集上运行本框架与竞争方法的全面对比。评估指标包括：Top-10通路重叠率（Jaccard相似系数）、差异代谢物AUPR（在有ground truth的spike-in子集上）、已知文献通路的捕获率（正向验证）和计算运行时间。

### 5.6 Ground Truth构建策略

**Phase 1 Ground Truth（注释正确性）：** 使用NIST标准品确定的化学结构（MSI Level 1），评估CP预测集是否以目标覆盖率包含正确注释。校准集的标准品注释由人工核查确认（对照CAS号、精确质量和MS2参考谱图）。

**Phase 2 Ground Truth（差异代谢物检测）：** 计算spike-in实验，在特征矩阵中对已知注释特征人工添加服从log-normal分布的差异（均值logFC=1，标准差0.3），以spike-in标签作为二分类ground truth评估AUPR。

**Phase 3 Ground Truth（通路分析可靠性）：** 采用跨队列一致性作为代理ground truth——若某通路在两个独立疾病队列中均获得Tier 1稳定性，将其视为生物学可信信号。同时使用文献记录的已知通路（糖尿病：糖酵解、TCA循环、支链氨基酸代谢；结直肠癌：色氨酸代谢、胆碱磷脂代谢）作为正向验证参照。

### 6.3 Discussion论证角度与预期Reviewer质疑回应

**预期质疑一："CP覆盖率保证需要exchangeability，但真实代谢物与标准品不具备exchangeability。"**

回应策略：承认局限，给出量化评估而非回避。在已知标准品的独立测试集上直接展示实际覆盖率偏差（预期在±5%以内），并讨论scaffold split如何通过在化学骨架层面隔离校准/测试集来最大化近似exchangeability。补充论证：Barber et al.（2023）的理论工作表明，CP在轻度协变量偏移（covariate shift）下仍能robustly控制coverage，对保守划分策略而言覆盖率超出比不足更可接受。最终将这一限制写入论文局限性部分，同时提出未来在类似化学空间的真实样本上构建大规模校准集的研究方向。

**预期质疑二："β的选择依赖于spike-in实验，而真实生物学中没有spike-in ground truth。"**

回应策略：直接承认，并从三个角度论证β选择的鲁棒性。其一，展示β在[1.0, 1.5]范围内对最终通路结论的敏感性分析（Supp Fig），证明在此区间内结论稳定；其二，展示三个独立数据集的最优β均落在相同区间，表明β具有跨数据集通用性；其三，从理论角度指出β=1是注释概率的无偏期望利用，任何接近1的β选择都有合理的理论依据，spike-in实验不过是验证这一理论直觉的实证工具。

**预期质疑三："贝叶斯通路模型需要MCMC，计算开销对常规实验室不可接受。"**

回应策略：给出精确计时基准（16核、200通路约3小时），并提供变分推断（ADVI）的快速近似选项，验证变分推断与完整MCMC的结论差异不超过5%且速度快10倍。论文主要图表使用完整MCMC结果以保证统计严格性，软件包同时提供快速近似模式作为实用选项，允许用户根据需求选择精度-效率权衡点。

**预期质疑四："与BAUM的区别仅是加了Phase 2的差异分析加权？"**

回应策略：这是最重要的差异化论证，需通过消融实验数据直接回答。BAUM的核心设计没有注释置信度加权的差异分析步骤，差异分析结果以等权重形式输入通路分析。本研究的Fig 6A消融实验直接展示：仅移除Phase 2（保留Phase 1和3）已导致可观察的结论差异，而完整三阶段传播的跨队列一致性指标超过任意两阶段组合，表明存在跨阶段的协同效应而非简单叠加。进一步，Phase 2的同分异构体信号分配提供了BAUM框架中不存在的通路-特征连接精化机制，是本框架独立的方法论贡献。

**预期质疑五："没有用真实IMS数据验证CCS贝叶斯更新。"**

回应策略：承认这是现阶段的方法局限。NIST标准品子集（约300-500个化合物具有CCS实测值，来自METLIN CCS数据库）作为有限但可用的验证数据集（Supp Fig），展示在有实测CCS的化合物上贝叶斯更新的实际效果。对无实测CCS的数据，理论分析和数值验证均表明CCS更新退化为弱信息先验（似然接近均匀分布），不引入系统性偏差。将完整TIMS-TOF数据集的CCS验证列为未来工作，并指出本框架已为此提供了基础设施（贝叶斯更新模块接受任意CCS精度的误差模型参数）。

## 七、脚本与软件产出

### 7.1 数据预处理层

**`convert_to_mzml.py`**：将原始厂商格式（.raw、.d）批量转换为标准化mzML，封装ProteoWizard msconvert命令行调用，输出文件名规范化和转换日志。预计约150行。

**`peak_detection.py`**：调用MZmine3 headless模式进行峰检测、对齐和gap-filling，管理batch XML配置的参数化生成，输出特征矩阵CSV和MS2谱图MGF。预计约200行。

**`annotate_candidates.py`**：对每个特征执行m/z宽窗口数据库检索和MS2候选预筛选，整合HMDB本地SQLite缓存和GNPS Faiss谱图索引，输出候选注释Parquet表（每行对应一个特征×候选对）。预计约350行。

**`predict_ccs.py`**：批量调用AllCCS2或DeepCCS模型对候选分子SMILES进行CCS预测，处理无效SMILES的异常情况，输出扩展后的候选注释Parquet表（含预测CCS列）。预计约120行。

**`build_hmdb_sqlite.py`**：解析HMDB XML文件构建本地SQLite数据库（含代谢物精确质量、SMILES、通路映射），支持增量更新和版本追踪。预计约250行。

**`build_spectral_index.py`**：基于ms2deepscore模型构建GNPS谱图库的Faiss向量索引，支持批量嵌入生成和索引持久化，输出可供近似最近邻检索的二进制索引文件。预计约180行。

### 7.2 CP核心算法层

**`compute_conformity_scores.py`**：实现四维conformity score计算，包括自适应权重逻辑回归学习和各维度的高斯核/余弦相似度计算。这是Phase 1的核心脚本，处理low-quality MS2的保守估计逻辑和跨化合物类别的权重适配。预计约500行。

**`calibrate_cp.py`**：在校准集上估计各目标覆盖率对应的分位数阈值，输出JSON格式的覆盖率-阈值映射表，支持scaffold split的化学分层校准分析。预计约180行。

**`generate_prediction_sets.py`**：用校准阈值对测试集每个特征生成概率预测集，计算边际候选概率和预测集大小统计，输出包含注释集合和概率分布的Parquet表。预计约220行。

**`validate_coverage.py`**：在有ground truth的标准品测试集上验证经验覆盖率，生成校准曲线图表和覆盖率偏差统计表，支持按化合物类别的分层验证。预计约180行。

**`bayesian_ccs_update.py`**：实现CCS贝叶斯后验更新，对有实测CCS的特征执行归一化后验计算，对无CCS数据的特征保持初始概率不变，处理无效CCS预测的弱信息先验退化。预计约200行。

**`isomer_disambiguation.py`**：识别预测集中含有同分异构体候选的模糊特征节点，构建同分异构体图结构，输出JSON格式的模糊节点索引。预计约150行。

### 7.3 加权差异分析层

**`prepare_weights.py`**：从CP预测集生成每个特征的注释概率权重，实现β参数的指数变换和权重归一化，输出供R脚本使用的特征权重CSV。预计约120行。

**`learn_beta.py`**：在spike-in实验数据上通过交叉验证搜索最优β参数，计算各β取值的AUPR评估指标，输出β-AUPR曲线数据和最优参数JSON。预计约250行。

**`run_limma_weighted.R`**：以precision weights注入limma线性模型执行加权差异分析，支持批次效应协变量、趋势方差建模和鲁棒eBayes估计，输出包含权重使用记录的完整差异分析结果。这是Phase 2的核心R脚本。预计约300行。

**`run_ihw.R`**：以注释置信度作为主协变量运行IHW多重检验校正，输出带IHW校正q值和分层权重的差异分析最终结果。预计约120行。

**`isomer_signal_split.py`**：对模糊特征节点按候选概率分配差异信号（logFC线性分配、t统计量平方根比例分配、p值概率加权公式），输出同分异构体拆分后的差异分析结果表。预计约200行。

**`benchmark_comparison.R`**：与等权重基准方法的对比分析脚本，计算AUPR、实际FDR和真阳性数量，生成FDR控制曲线和ROC曲线图表。预计约200行。

### 7.4 贝叶斯通路分析层

**`build_pathway_mapping.py`**：从HMDB/KEGG构建概率代谢物-通路成员关系映射，整合预测集的同分异构体模糊性，输出{通路ID: {化合物ID: 成员概率}}格式的JSON映射文件。预计约280行。

**`bayesian_pathway_scoring.py`**：Phase 3核心算法，实现分层贝叶斯通路活跃度模型：Beta先验参数的Empirical Bayes估计、注释概率加权的通路差异信号计算、PyMC NUTS采样推断，输出每个通路的后验均值、95%置信区间和贝叶斯因子。这是全流程最复杂的脚本。预计约600行。

**`monte_carlo_sensitivity.py`**：实现三层嵌套扰动（注释重采样700次+β扰动200次+校准集子集扰动100次），支持joblib并行加速，输出HDF5格式的完整扰动结果矩阵和Tier分类汇总CSV。预计约350行。

**`pathway_visualization.py`**：生成通路网络的交互式Plotly可视化（节点=通路，大小=代谢物数量，颜色=后验概率，边=共享代谢物）和用于出版的静态PDF图表。预计约220行。

### 7.5 端到端整合与可重现性

**`run_full_pipeline.py`**：Snakemake DAG的Python封装入口，通过YAML配置文件管理数据路径和超参数，提供断点续跑和日志记录，使完整流程的运行简化为单一命令。预计约200行。

**`compare_frameworks.py`**：在所有三个数据集上系统对比本框架与竞争方法，计算综合评估指标，输出全面基准测试结果表。预计约300行。

**`reproduce_figures.py`**：从最终结果文件重现论文全部主文图表（Fig 1-7）和补充图表，确保图表生成完全可重现，支持单独重生成任一图表。预计约500行。

**`Snakefile`**：Snakemake主流程文件，定义从原始数据到最终结果的完整DAG依赖关系（约40条规则），支持集群提交和本地运行两种模式。预计约400行。

**`config/config.yaml`**：超参数中央管理文件，包含Phase 1-3所有算法参数的默认值、数据路径配置和计算资源设置，所有参数带注释说明其来源和推荐调整范围。预计约150行。

## 九、里程碑时间线

**M1-2（1-2月）：基础设施建设**

计算环境配置，数据集获取（ST001050、MTBLS214、GNPS谱图库、HMDB数据库），谱图库索引建立，峰检测运行（MZmine3）并产出初始特征矩阵。产出：可用的候选注释Parquet表。

四维conformity score实现与调试，CP校准流程建立，实验P1-A/B/C完成，产出Phase 1评价报告。产出：经验证的prediction_sets_ccs_updated.parquet，CP校准曲线（Fig 2草稿）。

**M5-6（5-6月）：Phase 2核心实现**

limma precision weight注入实现，IHW协变量配置，β选择实验框架建立，同分异构体图构建模块实现。产出：核心R脚本和Python支持脚本，spike-in实验框架。

**M7-8（7-8月）：Phase 2完整验证**

实验P2-A至P2-D全部完成，与MassID等基准对比，代码重构和文档化。产出：Phase 2评价报告，Fig 3草稿，ST001050差异分析最终结果。

**M9-10（9-10月）：Phase 3实现与验证**

贝叶斯通路模型实现，MCMC调试，Monte Carlo三层扰动实现，实验P3-A/B完成。产出：通路贝叶斯打分模块，Fig 4草稿，Tier分类结果。

论文初稿完成，内部Reviewer预演，代码开源（GitHub + Zenodo DOI），软件包发布，论文提交。产出：投稿就绪的论文和可安装的开源软件包。
