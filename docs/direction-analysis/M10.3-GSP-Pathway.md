## 执行摘要

GSP-Pathway 方向将代谢物浓度变化作为代谢网络图上的图信号，利用图拉普拉斯谱分解实现通路层面的系统性分析。本调研结论：

- **技术可行性**: 中等。数学框架坚实，PyGSP 工具成熟，KEGG API 可支持网络构建，但代谢网络的特殊性（稀疏、有向、注释不完整）给 GSP 假设带来严峻挑战
- **创新性**: 真实存在。GSP 在代谢组学通路分析中确实几乎无先例，与 NetGSA/SPIA 存在本质区别
- **Nature Methods 级别判断**: 过于乐观。需要降级预期，Bioinformatics / PLoS Computational Biology 更合理，除非能在多个真实数据集上全面超越现有最优方法
- **最大风险**: 代谢网络稀疏性可能使图信号处理核心假设（平滑先验）失效

## 1. 技术详细分析

### 1.1 图信号处理数学基础

#### 图与图拉普拉斯

设代谢网络 $G = (V, E, W)$，其中：
- $V$：节点集，$|V| = n$，每个节点对应一个代谢物
- $E$：边集，边表示代谢反应中的共底物/产物关系
- $W$：权重矩阵，$w_{ij}$ 表示两代谢物间反应的关联强度

**度矩阵** $D = \text{diag}(d_1, ..., d_n)$，其中 $d_i = \sum_j w_{ij}$

**组合图拉普拉斯**（Combinatorial Laplacian）：
$$L = D - W$$

**归一化图拉普拉斯**（PyGSP 默认使用）：
$$\mathcal{L} = D^{-1/2} L D^{-1/2} = I - D^{-1/2} W D^{-1/2}$$

归一化拉普拉斯特征值范围为 $[0, 2]$，避免了节点度差异带来的数值问题。

#### 图谱分解

对 $\mathcal{L}$ 进行特征分解：
$$\mathcal{L} = U \Lambda U^T$$

其中 $U = [u_1, u_2, ..., u_n]$ 为特征向量矩阵（图傅里叶基），$\Lambda = \text{diag}(\lambda_1, ..., \lambda_n)$，$0 = \lambda_1 \leq \lambda_2 \leq ... \leq \lambda_n$。

**图傅里叶变换**：
$$\hat{f} = U^T f$$

其中 $f \in \mathbb{R}^n$ 是图信号（代谢物浓度变化向量），$\hat{f}$ 是其频谱表示。

#### 低通滤波的生物学含义

低频分量（小特征值对应的特征向量）在网络中变化缓慢，对应代谢物在通路内的系统性协同变化。高频分量对应局部、剧烈的跳变，可能反映噪声或孤立的代谢变化。

**低通滤波器**（如 PyGSP 示例中的 $g(\lambda) = \frac{1}{1+\tau\lambda}$）：

$$f_{\text{smooth}} = U g(\Lambda) U^T f = U \text{diag}(g(\lambda_1), ..., g(\lambda_n)) U^T f$$

通路层面的通量变化——整条通路系统性上调/下调——在图中表现为低频信号，因为通路内部代谢物高度互联、协同变化。这是 GSP-Pathway 的核心假设。

#### 图信号平滑度指标

Dirichlet 能量（图全变差）：
$$S(f) = f^T L f = \sum_{(i,j) \in E} w_{ij}(f_i - f_j)^2$$

$S(f)$ 越小，信号越平滑，通路内代谢物变化越协调。可以用此度量通路活性：平滑度高的频率分量代表通路系统性变化。

### 1.2 代谢网络作为图的建模方式

**节点定义选项**：

| 建模方案 | 节点 | 边 | 优劣势 |
|---------|-----|---|--------|
| 方案A（代谢物图） | 代谢物 | 共享反应的代谢物对之间连边 | 最直观，与测量直接对应；问题：辅因子（ATP、NADH）造成超高度节点（hub） |
| 方案B（二部图） | 代谢物 + 反应 | 代谢物参与反应则连边 | 结构完整；问题：维度增加，GSP 直接映射复杂 |
| 方案C（压缩代谢物图） | 代谢物（过滤辅因子） | 方案A + 移除高度节点 | 推荐用于 GSP |

**KEGG 数据来源**：
- `rest.kegg.jp/link/reaction/compound`：化合物-反应二部图（tab 格式，cpd:CXXXXX rn:RXXXXX）
- `rest.kegg.jp/link/cpd/map{ID}`：特定通路的代谢物列表
- KGML 格式：含方向信息的 XML 格式通路图
- 人类代谢通路：~95条核心代谢通路（hsa 前缀）

**边权重定义选项**：
- 二值（0/1）：最简单，但丢失信息
- 反应 stoichiometry 系数：更精确
- 共现频率（在多条通路中同时出现）：反映代谢物连接重要性

### 1.4 与现有方法的理论比较

#### ORA（Over-Representation Analysis）

**方法**：超几何检验，问显著代谢物是否富集于某通路。

**核心问题**：
1. 需要 arbitrary 显著性阈值划分"显著"/"不显著"
2. 通路独立性假设（忽略通路间共享代谢物）
3. 完全忽略网络拓扑
4. 完全忽略通路内部代谢物的变化量级和方向

**GSP-Pathway 如何改进**：所有代谢物均参与计算，无需阈值；通路内邻接代谢物的一致性变化被图信号平滑度捕获。

#### MSEA（Metabolite Set Enrichment Analysis）

**方法**：借鉴 GSEA，对代谢物按变化量排序，计算通路内代谢物是否集中分布于排名极端区域（Kolmogorov-Smirnov 统计量）。

**核心问题**：同样视通路为独立代谢物集合，不考虑网络拓扑；计算通路得分时代谢物等权。

**GSP-Pathway 如何改进**：网络中心位置的代谢物（高度节点）在图信号中自然获得更大权重；邻接代谢物的协同变化可以被低频分量捕获。

#### Mummichog

**方法**：不依赖先验代谢物注释，从 LC-MS 特征表直接利用代谢网络拓扑做置换检验，预测功能活性。关键创新：绕过代谢物鉴定步骤，用网络"guilt by association"原则。

**与 GSP-Pathway 的关系**：两者都利用代谢网络拓扑，但层次不同：
- Mummichog：利用拓扑判断未知特征是否"可能属于活跃通路"（解决注释问题）
- GSP-Pathway：利用拓扑对已注释代谢物的浓度变化做频谱分析

**互补性**：可以考虑先用 Mummichog 做注释，再用 GSP-Pathway 做通路分析。

#### SPIA（Signaling Pathway Impact Analysis）

**方法**：结合 ORA 证据和信号扰动分析（PertFactor），利用通路内有向图上的基因/代谢物变化的下游传播效应计算扰动得分。

**数学框架**（Ma et al., 2016，Bioinformatics）：
$$\text{TA} = \Delta E + \sum_{\text{upstream}} \text{PertFactor} \times \Delta E_{\text{upstream}}$$

**与 GSP-Pathway 的本质区别**：
- SPIA 是基于有向传播的扰动模型，关注"谁影响了谁"
- GSP-Pathway 是频谱分析，关注"变化模式的频率结构"
- SPIA 是确定性信号传播，GSP-Pathway 是统计频谱分析

#### NetGSA（Network-based Gene Set Analysis）

**方法**（Ma, Shojaie, Michailidis，Bioinformatics 2016 + PLoS Comput Biol 2021）：
在线性混合效应模型框架中，将网络拓扑（图 Laplacian 正则化项）嵌入检验统计量，同时利用数据估计网络和已知网络信息，实现路径层面的联合检验。

$$g(Y) = X\beta + h(ZL_R) + \epsilon$$

其中 $L_R = (I + \tau \tilde{L})^{-1}$ 为正则化归一化 Laplacian（与 PyGSP 低通滤波在数学上等价！）。

**与 GSP-Pathway 的本质区别**：
- NetGSA 是假设检验框架（统计显著性），GSP-Pathway 是频谱分解框架（信号分析）
- NetGSA 已有成熟 R 包（netgsa on CRAN），有实证验证
- GSP-Pathway 的频谱视角提供了 NetGSA 没有的：哪些频率分量主导了通路变化，高频噪声如何与低频生物学信号分离

**核心竞争关系分析**：NetGSA 是 GSP-Pathway 最大的概念竞争者。两者都用拉普拉斯对网络拓扑建模，但 GSP-Pathway 提供了更丰富的频谱解释层。如果不能清晰区分并在 benchmark 上显示优势，将很难在高影响力期刊发表。

### 1.5 代谢物注释不完整的处理

**问题严重性**：典型非靶向代谢组学鉴定率约 10-30%，大量 LC-MS 特征无法映射到 KEGG 化合物节点。

**处理方案**：

| 方案 | 描述 | 优劣势 |
|------|------|--------|
| 方案1: 子图分析 | 仅对已注释代谢物构建子图 | 最保守；图稀疏性加剧 |
| 方案2: 图正则化 | 用 $L_R = (I + \tau L)^{-1}$ 平滑稀疏图（PaIRKAT 方案） | 有理论支撑，可处理缺失节点 |
| 方案3: 图补全 | 用化学结构相似性推断未注释代谢物的网络位置 | 创新性高，但引入额外假设 |
| 方案4: Mummichog 前置 | 先用 Mummichog 扩展注释，再 GSP 分析 | 可行的工程方案 |

**PaIRKAT 证据**（Carpenter et al., PLoS Comput Biol 2021）：模拟实验显示正则化方法对 70% 缺失边和 25% 缺失节点仍保持统计功效，但这是在基因层面（稀疏程度远低于代谢组学）验证的结果。

## 2. 文献证据

### 2.1 GSP 在生物信息学中的已有应用

**系统综述（最重要参考）**：
- Li R et al. (2023). "Graph Signal Processing, Graph Neural Network and Graph Learning on Biological Data: A Systematic Review." *IEEE Review of Biomedical Engineering*. — 确认 GSP 在生物医学领域有一定应用，但综述范围包括神经影像、基因调控网络，**无代谢组学通路分析应用**。

**已有 GSP 生物应用**：
1. **神经影像/连接组**：Petrovic et al. (2019). "Guided graph spectral embedding: Application to the *C. elegans* connectome." *Network Neuroscience*. — 将脑功能信号投影到结构连接组图上做频谱分析。
2. **单细胞转录组**：Venkat et al. (2024). "Mapping the gene space at single-cell resolution with gene signal pattern analysis (GSPA)." *Nature Computational Science*. — 用扩散小波字典学习基因表达模式（*重要参考：这是 GSP 在 omics 数据中最高级别的发表*）。
3. **基因调控网络推断**：scSGL (2022, Bioinformatics)：用图信号处理框架从单细胞数据推断有符号基因调控网络。
4. **图拉普拉斯正则化**：Liu et al. (2022, J Theor Biol)：图拉普拉斯正则化用于 Alzheimer 生物标志物识别。

**关键发现**：GSP 在**代谢组学通路分析**中确实几乎空白，GSPA（Nature Computational Science, 2024）提供了最强的方法论先例，可作为核心对标。

### 2.2 代谢组学通路分析现有方法综述

**MetaboAnalyst 支持的方法**（Chong et al., 2019）：
- ORA（Over-Representation Analysis）：最基础，超几何检验
- MSEA（Metabolite Set Enrichment Analysis）：~6300 代谢物集，支持富集分析
- GSEA：基于排序的富集，不需要阈值
- Mummichog（v2）：网络拓扑 + 置换检验，支持未鉴定特征
- 拓扑分析：MetaboAnalyst 的拓扑模块考虑通路内代谢物的中心性

**方法分代**：
- 第一代（2000s）：ORA — 忽略量级、忽略拓扑
- 第二代（2010s）：GSEA/MSEA — 利用量级，忽略拓扑
- 第三代（2012-）：SPIA、Mummichog、NetGSA — 利用网络拓扑
- 第四代（潜在）：GSP-Pathway — 频谱分析视角，将拓扑作为信号变换算子

### 2.3 网络拓扑在通路分析中的已有尝试

**SPIA**（Tarca et al., Bioinformatics, 2009）：
- 最早引入通路有向拓扑的方法之一
- 原始设计用于基因组学，代谢组学适配有限
- 核心：ORA + 基于有向图的扰动因子

**NetGSA**（Ma, Shojaie, Michailidis）：
- 2016 Bioinformatics: 方法原文，线性混合模型 + 网络正则化
- 2019 BMC Bioinformatics: 拓扑富集分析方法比较研究（NetGSA vs SPIA vs gsea）
- 2019 Bioinformatics: 慢性肾病脂质通路差异网络富集分析（实际应用）
- 2021 PLoS Comput Biol: netgsa R 包（KEGG/Reactome 集成，C++ 加速，Cytoscape 可视化）

**PaIRKAT**（Carpenter et al., PLoS Comput Biol 2021）：
- 核心创新：图正则化 Laplacian $(I + \tau\tilde{L})^{-1}$ 嵌入核机器回归
- 专为代谢组学设计（PaIRKAT 在 COPD 代谢组数据上验证）
- 对 70% 缺失边仍鲁棒
- **与 GSP-Pathway 数学基础高度重叠，是最直接的竞争/先驱工作**

**MetaMapp**（Barupal et al., BMC Bioinformatics 2012）：
- 化学相似性 + 生化通路信息整合做代谢组学可视化

## 3. 具体执行路径和实验方案

### 3.1 KEGG/Reactome 代谢网络构建

**数据获取**（完整流程）：

```python
import requests
import networkx as nx
import numpy as np
from scipy.sparse import csr_matrix

# Step 1: 获取 KEGG 人类代谢通路列表
def get_kegg_pathways():
    resp = requests.get("https://rest.kegg.jp/list/pathway/hsa")
    # 过滤出代谢通路（hsa00xxx, hsa01xxx）
    pathways = {}
    for line in resp.text.strip().split('\n'):
        pid, name = line.split('\t')
        if any(pid.startswith(f'hsa{i}') for i in ['00', '01']):
            pathways[pid] = name
    return pathways

# Step 2: 构建化合物-反应二部图
def build_compound_reaction_graph():
    resp = requests.get("https://rest.kegg.jp/link/reaction/compound")
    G = nx.Graph()
    for line in resp.text.strip().split('\n'):
        cpd, rxn = line.split('\t')
        G.add_node(cpd, type='compound')
        G.add_node(rxn, type='reaction')
        G.add_edge(cpd, rxn)
    return G

# Step 3: 投影为化合物-化合物图（共享反应）
def project_to_compound_graph(bipartite_G):
    compounds = [n for n, d in bipartite_G.nodes(data=True) if d.get('type') == 'compound']
    return nx.bipartite.projected_graph(bipartite_G, compounds)

# Step 4: 过滤辅因子高度节点（ATP, NADH, CoA 等）
COFACTORS = {'cpd:C00002', 'cpd:C00008', 'cpd:C00003', 'cpd:C00004',
             'cpd:C00010', 'cpd:C00001', 'cpd:C00013', 'cpd:C00009'}
def filter_cofactors(G, max_degree=None):
    if max_degree:
        to_remove = [n for n, d in G.degree() if d > max_degree]
    else:
        to_remove = COFACTORS
    G.remove_nodes_from(to_remove)
    return G
```

**网络规模预估**：
- KEGG 全局代谢物：~18,000 化合物，但有代谢通路注释的约 3,000-4,000
- 过滤辅因子后：~2,000-3,000 节点
- 边数：~10,000-30,000（取决于通路覆盖）
- 连通分量：多个（这是核心问题，见压力测试）

### 3.2 图信号定义

**推荐方案（多信号对比）**：

| 信号定义 | 公式 | 优势 | 劣势 |
|---------|------|------|------|
| log Fold Change | $f_i = \log_2(\text{FC}_i)$ | 直接反映浓度变化方向和幅度 | 需要精确定量 |
| Signed -log10(p) | $f_i = \text{sign}(\text{FC}_i) \cdot (-\log_{10} p_i)$ | 综合显著性和方向 | p 值有多检验问题 |
| Rank-based | $f_i = \text{rank}(\text{FC}_i) / n$ | 鲁棒，无分布假设 | 损失量级信息 |
| z-score | $f_i = (x_i - \mu_x) / \sigma_x$ | 标准化，便于比较 | 假设正态 |

**未注释代谢物处理**：将其节点赋值为 0（中性），或用 Mummichog 预测后赋值。

### 3.3 低频/高频分量的生物学解释

| 频率分量 | 数学特征 | 生物学解释 | 举例 |
|---------|---------|-----------|------|
| 低频（$\lambda$ 小，特征向量平滑） | 网络内变化缓慢 | 整条通路系统性上调/下调 | 三羧酸循环统一下调 |
| 中频 | 通路间边界的信号跳变 | 局部代谢重编程 | 糖酵解上调但 TCA 不变 |
| 高频（$\lambda$ 大，特征向量跳变） | 相邻节点符号相反 | 噪声 / 个别代谢物特异性变化 | 单个代谢物的测量噪声 |

**通路得分计算**：

对于通路 $P$（节点子集），提取低频滤波后的信号 $f_{\text{low}}$：
$$\text{PathwayScore}(P) = \frac{1}{|P|} \sum_{i \in P} |f_{\text{low},i}|$$

或基于 Dirichlet 能量的一致性得分：
$$\text{Coherence}(P) = 1 - \frac{\sum_{(i,j) \in E(P)} w_{ij}(f_i - f_j)^2}{|P| \cdot \text{Var}(f_P)}$$

## 7. 创新点分析

### 真实创新点

1. **频谱视角（核心创新）**：将代谢通路分析问题重新表述为图信号频谱分析问题。低频/高频分量的分离提供了 NetGSA 和 SPIA 无法直接给出的信息：系统性通路变化（低频）vs. 局部/噪声变化（高频）的区分。

2. **通路间关联分析**：代谢物在多条通路中共享（如 pyruvate 在糖酵解和 TCA 循环间），GSP 的图信号在全局代谢网络上传播，天然捕获通路间的串扰（cross-talk），而 ORA/MSEA 将通路视为独立集合。

3. **连续信号视角**：无需任意阈值划分显著/不显著，全部代谢物参与分析，信号连续。

4. **信号分离的生物学解释**：低频分量 = 通路层面协调变化，高频分量 = 局部扰动，这一解释在概念上清晰，可为临床决策提供新信息。

### 衍生创新（可扩展方向）

- **多组学整合**：将代谢组学信号和转录组信号同时定义在代谢-基因异质图上
- **时序分析**：定义时变图信号（动态代谢重编程）
- **患者分层**：单样本级别通路频谱作为特征做聚类

### 创新弱点（压力测试中详述）

- 频谱分析框架本身不新——NetGSA 已用 Laplacian 正则化，PaIRKAT 也是如此
- "图傅里叶分析"在生物信息学中已有 GSPA（Nature Computational Science, 2024）

## 9. 压力测试（关键风险评估）

### 压力测试 9.1：代谢网络连通性问题

**问题陈述**：KEGG 完整代谢网络约 3000 化合物，但典型非靶向代谢组学实验：
- 检测到的特征：1,000-10,000 个 m/z 特征
- 置信鉴定（Level 1-2）：100-500 个代谢物
- KEGG 映射成功：50-300 个

**实际场景中的图结构**：
- 50-300 个节点，在 3000 节点的全局网络中只有 10-30% 覆盖
- 大量孤立节点（无鉴定代谢物的网络区域）
- 图会断裂成多个连通分量

**对 GSP 的影响**：
- 图谱分解在断裂图上没有意义：每个连通分量独立，低频分量退化为分量内常数向量
- 若代谢组注释覆盖的区域恰好是稀疏连接区，则信号平滑度假设完全失效
- 极端情况：300个已鉴定代谢物构成 50 个小连通分量，每个分量平均 6 个节点——无法做有意义的谱分析

**严重程度**: 高。这是 GSP-Pathway 最核心的技术风险，直接威胁方法可行性。

**缓解方案（按可行性排序）**：
1. 限定使用靶向代谢组学数据（注释完整性高，150-500 个代谢物，覆盖特定代谢通路，图连通性好）— 限制了方法的应用范围
2. 图补全：用化学结构相似性（如 Tanimoto coefficient on Morgan fingerprints）为未鉴定特征推断节点位置 — 引入额外假设
3. 分析连通最大子图（largest connected component）并报告覆盖率 — 合理但需谨慎解释
4. 借鉴 PaIRKAT 的图正则化方案：$(I + \tau L)^{-1}$ 可以在稀疏图上自动"平滑"缺失连接 — 但这实际上是另一种方法而非真正的 GSP

### 压力测试 9.2：图信号平滑假设在代谢网络上是否成立

**假设核心**：通路层面的系统性变化应该表现为低频信号（网络邻接节点协同变化）。

**反驳论点**：
1. **代谢网络不是通信网络**：图信号处理的低通假设来自物理/通信信号（时间或空间上的连续信号在图拓扑上平滑传播）。代谢网络中，代谢物 A 产生代谢物 B 不等于 A 和 B 的浓度变化一定相关——酶活性、转运体、细胞区室都可以解耦两者。
2. **代谢物并非 freely diffusing signal**：mRNA 表达在细胞核连续扩散，相邻基因表达相关性有物理基础。代谢物在不同反应中浓度变化受酶调控，与网络邻接性的相关性弱于基因/蛋白质网络。
3. **经验证据**：无直接证据支持代谢浓度变化在代谢网络上具有低频特性。这是 GSP-Pathway 需要在方法论上首先验证的核心假设。

**评估**：这是 GSP-Pathway 的根本假设风险。如果代谢物变化在代谢网络上不具有低频结构，则整个频谱分析框架失去生物学意义，退化为一种特殊的加权汇总方法。

**验证方案**：
- 用多个公开数据集，计算已知活跃通路的代谢物变化向量的 Dirichlet 能量（平滑度指标）
- 与随机排列的代谢物标签对比（零假设：代谢物变化在网络上没有空间结构）
- 如果真实数据的 Dirichlet 能量显著低于随机，则假设成立

**执行建议**：在方法论第一步必须包含这个验证，否则是没有前提假设验证的方法。

### 压力测试 9.3："Nature Methods 级别"判断是否过于乐观

**Nature Methods 发表标准**：
1. 方法在概念上解决了此前无法解决的问题
2. 在多个大型、多样化数据集上系统验证优于现有最优方法
3. 方法对领域产生实质性影响（被广泛采用）
4. 软件/代码质量达到工具包水平（可安装、有文档、可重现）

**GSP-Pathway 当前差距**：

| 标准 | 现状 | 差距 |
|------|------|------|
| 概念新颖性 | 代谢组学中确实新颖 | 频谱工具本身不新；NetGSA/PaIRKAT 存在 |
| 解决无法解决的问题 | 目前未见明确案例 | 需要找到 ORA/NetGSA 均失败但 GSP 成功的场景 |
| 系统验证 | 尚未开始 | 需 5+ 数据集，2+ 物种 |
| 代码质量 | 尚未开发 | 需要完整 Python 包 |

**诚实评估**：Nature Methods 目标过于乐观，尤其在 PaIRKAT（2021, PLoS Comput Biol）和 GSPA（2024, Nature Computational Science）已发表的背景下。

**合理期刊目标**：
- 首选：Bioinformatics（牛津，高影响力方法学期刊）
- 次选：PLoS Computational Biology
- 高目标：Briefings in Bioinformatics / Nucleic Acids Research
- 如果在单细胞代谢组学或空间代谢组学的新兴场景中有突破：考虑 Nature Methods（但需更强数据）

### 压力测试 9.4：与 NetGSA/SPIA 的本质区别是什么

**与 NetGSA 的数学关系**：

NetGSA 中的网络正则化项 $L_R = (I + \tau\tilde{L})^{-1}$ 与 PyGSP 中的低通滤波器 $g(\lambda) = \frac{1}{1+\tau\lambda}$ 在数学上完全等价——都是图 Laplacian 的谱域低通变换。

**根本区别**：
- NetGSA 的目标是**假设检验**（通路差异是否显著，给出 p 值）
- GSP-Pathway 的目标是**信号分解**（通路变化的频率结构分析，给出频谱）

**实际意义**：
- NetGSA 输出：哪些通路显著（p < 0.05），是/否
- GSP-Pathway 输出：每条通路的变化是系统性低频变化还是局部高频变化，以及各频率分量的贡献

**这个区别够不够发论文**：

勉强够，但需要强调：
1. 频谱分解提供了 NetGSA 没有的**解释性**（不只是"通路是否显著"，而是"通路变化的频率特征"）
2. 需要展示频谱特征在生物学解释上提供了额外信息（如识别通路间串扰、区分系统性 vs. 局部扰动）
3. 如果仅在通路排名/发现上比 NetGSA 更好，差距不足以支撑顶刊

**与 SPIA 的本质区别**：

明确且大：SPIA 是有向扰动传播模型（谁扰动了谁），GSP-Pathway 是频谱分析（变化的频率结构）。两者方向完全不同，竞争关系小于互补关系。

### 压力测试 9.5：辅因子问题

**问题**：水（C00001）参与 KEGG 中约 5,000+ 反应，ATP（C00002）、NADH 等参与数百反应。在化合物-化合物图（共享反应）中，这些辅因子将成为极高度节点，度数可能达到 2000+。

**对谱分析的影响**：超高度节点主导低频特征向量，导致几乎所有代谢物与水/ATP"相关"，低频分量无生物意义。

**缓解**：必须过滤辅因子。但过滤名单需要领域专家确认，边界模糊（如 glutamate 既是辅因子又有重要代谢意义）。

## 11. 建议与结论

### 建议执行策略

**首要任务（第1个月，做还是放弃的决策点）**：

验证核心假设：在 2-3 个公开代谢组学数据集上计算已知活跃通路的 Dirichlet 能量，对比随机排列。如果代谢物变化在代谢网络上不比随机更平滑（p > 0.05），则方向存在根本性问题，应重新评估。

**如果假设验证通过**：

1. 专注于靶向代谢组学应用场景（注释完整性高），避免稀疏图问题
2. 明确定位：GSP-Pathway 不是替代 NetGSA，而是提供额外的频谱解释层
3. 与 GSPA（Nature Computational Science, 2024）对齐叙述框架：同样是"将组学数据作为图上的信号"，但针对代谢通路分析
4. 在空间代谢组学或单细胞代谢组学（新兴场景）中寻找特别突出的应用

**期刊目标调整**：Bioinformatics 或 PLoS Computational Biology（6-8 IF），如有突破性应用再考虑冲击 Nature Methods。

### 核心优先级列表

1. **验证低频假设**（必须，决策性）
2. **辅因子过滤策略**（必须，基础工程）
3. **网络连通性处理方案**（必须，核心挑战）
4. **与 NetGSA 的差异化定义**（必须，发表前提）
5. **benchmark 设计**（重要，多数据集）
6. **频谱可视化**（加分项，提升方法可用性）
