# Findings — MetaboFlow 质谱数据库建设

## Key Decisions

1. **Level 3 从原始来源重建**：增量 append 导致数据损坏（FooDB 全部精确质量错误、HMDB 87% 丢失），改用 rebuild_level3.py 一次性重建
2. **FooDB CSV 列偏移**：description 字段含未引号化逗号，csv.DictReader 也无法正确解析。改用列索引（col9=mass, col10=InChIKey）绕过
3. **T3DB InChIKey 前缀**：字段值带 `InChIKey=` 前缀，validate_inchikey() 增加前缀剥离
4. **Lotus 28K 仅标识条目**：PubChem 中不存在，保留为 ID-only 条目（不参与 MS1 匹配，MS2 命中后提供物种上下文）
5. **HMDB predicted 碰撞能量**：原始文件无元数据，基于 CFM-ID 标准行为推断 10/20/40 eV（已在文档中明确标注）
6. **ISDB negative 格式**：原始 MGF 直接复制未转换，修复后用 RDKit 从 InChI 计算完整 InChIKey
7. **去重保留来源追溯**：每条去重后的谱图保留 Sources 字段记录所有数据库来源

## Important Context

- 最接近的竞品 FragHub（Anal Chem 2024）整合 4 来源 / 32K 化合物，MFSL 超过 30 倍
- Dorrestein 团队（Nature 2024）的 Reverse Metabolomics 方法与用户的组合化学建库方案直接相关
- NIST 23 v23 已禁用 MSP 导出，$550 买了也无法集成到 matchms；v20 可导出但已停售
- mzCloud 条款明确禁止批量下载

## Rejected Approaches

- **增量 append 构建 L3**：多次导致数据损坏、格式混乱，改为 rebuild 脚本一次性重建
- **Docker + Wine 转 NIST 库**：ARM64 兼容性失败，改由学生在 Windows 上操作
- **awk 解析 FooDB CSV**：列偏移无法修复，改用 Python csv.reader + 列索引
- **MoNA 全库下载**：链接过期需认证，用 GNPS-MONA 镜像替代
